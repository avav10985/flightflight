## RuView 是什麼

**用 WiFi 訊號當「雷達」做空間感知** —— 不用相機、不用穿戴裝置，**穿透牆壁**偵測人體、量心跳呼吸、估姿態。

## 核心原理

每台 WiFi 路由器都在發射電磁波。人類動作、呼吸、甚至靜坐都會擾動這些波。RuView 用 ESP32-S3 抓取 **CSI（Channel State Information，通道狀態資訊）**，把擾動轉換成可分析的資料。

## 能做的事

| 功能 | 效果 |
|------|------|
| 👤 存在偵測 | 穿牆抓人、計數、進出記錄 |
| 🫁 呼吸監測 | 6~30 BPM 非接觸量測 |
| 💓 心跳監測 | 40~120 BPM 非接觸量測 |
| 🦴 姿態估計 | 17 個關鍵點（COCO 標準）|
| 🚶 動作辨識 | 走路 / 坐 / 跌倒偵測 |
| 🏠 環境指紋 | 識別房間、偵測物品變動 |
| 😴 睡眠分析 | 睡眠分期、呼吸中止篩查 |
| 🧱 穿牆距離 | 最遠 5m |

## 硬體

- 最低 **ESP32-S3 × 1**（約 $9 USD）→ 基本感測
- **3~6 顆 ESP32-S3 組網狀**（~$54）→ 完整功能
- **+ Cognitum Seed 持久記憶模組**（~$140）→ 加 AI 整合、kNN 搜尋、加密見證鏈

⚠️ **不支援原版 ESP32 和 ESP32-C3**（單核 CPU 算力不夠跑 CSI DSP）

## 技術亮點

1. **多頻段網狀**：在 6 個 WiFi 通道間跳頻，把鄰居路由器當「免費雷達源」
2. **脈衝神經網路**：30 秒內學會新環境
3. **邊緣運算**：完全在 ESP32 上跑，**不需要雲端、不需要網路**
4. **加密見證**：每筆量測都有 Ed25519 簽章
5. **雙語言實作**：Rust（v2）+ Python（v1）

但這是「整合兩個專案」的進階題，**不在你目前的目標範圍**。先把基本飛控做穩比較實際。

| 雙目視覺 | 立體深度圖 | 30~100ms | 進階 |

## 唯一能勉強搭上的場景

```

## 結論

**繼續用 VL53L1X**。RuView 是有趣的研究專案，但不是飛機避障的解法。

### 🤖 Claude `2026-05-17 16:15:09`

好問題。CSI 感測**整套理論建立在「感測器靜止、目標移動」**這個假設上。移動節點會破壞所有底層數學。具體有 5 個致命原因：

## 1. 自身運動 vs 環境運動「無法區分」

CSI 偵測的是「**多徑訊號的變化**」：
- 牆靜止、人在動 → 變化來自「人」→ 抓到目標 ✅
- **無人機在飛、牆靜止 → 變化看起來也是「牆相對於我在動」→ 系統以為整面牆是個移動目標** ❌

從 CSI 數據看，「**我動 1 公尺**」和「**目標動 1 公尺**」**長得一模一樣**，無法用演算法分離。

## 2. 都卜勒效應淹沒目標訊號

CSI 用都卜勒頻移抓微小運動：
- **人類呼吸**：胸腔起伏 5mm → 都卜勒 ~0.5 Hz
- **心跳**：血管搏動 0.5mm → 都卜勒 ~1.5 Hz
- **無人機飛 1 m/s** → 都卜勒 **約 17 Hz**

無人機自身運動的都卜勒**比目標訊號大 30 倍以上**，把所有有用訊號淹掉。

## 3. 環境基線（Baseline）持續失效

CSI 感測必須先「**學習環境基線**」（記住沒人時的多徑指紋），之後比較變化才能偵測目標：
```
基線記憶：感測器在 A 點 → CSI 指紋 X
飛機飛到 B 點（1m 外）→ CSI 指紋變 Y（完全不同的多徑路徑）
→ 系統以為「環境劇變了」→ 警報亂響或乾脆停擺
```

要解決就得**每 0.1 秒重新學一次基線**，但 RuView 學一次要 30 秒。

## 4. 多節點幾何分集（Geometric Diversity）崩潰

RuView 多節點定位用的是**已知固定座標**做三角測量：
```
節點 A (x=0,   y=0)   ─┐
節點 B (x=3,   y=0)   ─┼─→ 已知幾何 → 解方程式 → 目標位置
節點 C (x=1.5, y=2.5) ─┘
```

無人機是**單一節點且座標一直變**，幾何方程式無解。退化成「**只能感知有東西在動，但不知道在哪**」，比 PIR 紅外感測器還不如。

## 5. 馬達震動 + 旋轉破壞相位同調

CSI 用「**相位資訊**」（不只是訊號強度）抓微動：
- 4 顆馬達 5000~10000 RPM 震動 → 天線相位噪音
- 機身偏航 / 翻滾 → 天線方向圖旋轉 → 相位重置
- 飛機加速 / 減速 → 都卜勒不穩定

CSI 需要的相位精度是 **<1°**，無人機環境根本達不到。

## 類比說明

| 場景 | 比喻 |
|------|------|
| RuView 固定節點 | 你坐在安靜圖書館，能聽到隔壁桌的細語（人類呼吸） |
| 移動節點（你的無人機）| 你在馬路上邊跑邊吼，還想聽到隔壁桌的細語 |

訊號量級差太多，物理上不可能。

## 為什麼雷達/光達在移動載具上沒這問題

「主動式」感測器（雷達、光達、超音波、雷射 ToF）的工作方式完全不同：
- **自己發射**訊號，**自己收回波**
- 「飛行時間（Time of Flight）」是絕對量，跟感測器運動無關
- 距離 = 光速 × 時間 / 2，不受都卜勒、多徑、環境基線影響

所以光達、雷達裝在無人機上**完全沒問題**。CSI 屬於「被動式無線環境感測」，**根本不是為移動載具設計的**。

## 一句話總結

> CSI 感測能用，是因為「**參考框架（reference frame）穩定**」。把感測器搬到無人機上 = 把參考框架打爛 = 所有演算法假設崩潰。

## MRI vs WiFi CSI 根本差異

| 項目 | MRI | WiFi CSI |
|------|-----|---------|
| 物理原理 | 氫原子核磁共振（NMR）| 電磁波散射 / 多徑 |
| 頻率 | 數十 MHz 射頻 + **強磁場（1.5~7T）** | 2.4 / 5 GHz 微波 |
| 能進入身體嗎 | ✅ 可以激發體內氫核 | ❌ 微波被體表水分大量吸收（衰減 ~30dB/cm） |
| 解析度 | **<1 mm** | **公尺級**（受波長 12cm 限制） |
| 能區分組織 | ✅ 脂肪/肌肉/腦灰質白質都不同 | ❌ 只能看到「體積大致在哪」 |
| 能看內臟 | ✅ 是主要用途 | ❌ 完全看不到 |
| 設備 | 數千萬台幣、需冷卻液氦 | 一顆 ESP32 USD$9 |

## 物理限制：為什麼 WiFi 看不進身體

**衍射極限（Diffraction Limit）**：
- 任何成像系統的解析度上限 ≈ 波長的一半
- WiFi 2.4 GHz 波長 = 12.5 cm → 解析度極限 **≈ 6 cm**
- 連肋骨間距（~3cm）都解析不出來，更別說細部結構

**穿透深度（Penetration Depth）**：
- 人體 70% 是水
- 水對 2.4 GHz 微波**吸收極強**
- 大約進入皮膚 1~2 cm 訊號就衰減超過 99%
- 內部組織反射回來的訊號根本回不到接收器

## 但 WiFi CSI 可以做這些「擬醫療」功能

| 能力 | 原理 | 醫療相關性 |
|------|------|----------|
| 呼吸監測 | 胸腔起伏 5mm → 都卜勒 | ✅ 睡眠呼吸中止症篩查 |
| 心跳監測 | 心搏震動傳到體表 | ✅ 非接觸生命徵象 |
| 跌倒偵測 | 動作姿態變化 | ✅ 長照、獨居老人 |
| 睡眠分期 | 整夜動作 + 呼吸模式 | ✅ 睡眠品質評估 |
| 步態分析 | 走路時 CSI 變化 | ✅ 帕金森氏症初期偵測 |

**這些都靠「體表反射」做的，沒有真正「看穿身體」**。屬於遠端生命徵象監測（remote vital sign monitoring），不是醫學成像。

## 真的有「微波成像」存在嗎？

有，但**不是 WiFi 等級**：

| 技術 | 頻率 | 用途 |
|------|------|------|
| **mmWave 雷達** | 24~77 GHz | 機場全身掃描儀（看衣服底下） |
| **微波乳房成像** | 1~15 GHz | 早期乳癌篩檢研究（有限商品化） |
| **THz 成像** | 0.1~10 THz | 安檢、皮膚癌、牙科 |
| **超寬頻雷達 UWB** | 3~10 GHz | 心跳/呼吸非接觸監測商品 |

這些用**比 WiFi 高很多的頻率**換解析度。**但都不能取代 MRI 看內臟**，最深也只能看皮膚下 1~2cm（高頻穿透更差）。

## 真正取代 MRI 的非游離輻射方案

要無輻射又能看內臟，目前只有：

| 技術 | 原理 | 適用 |
|------|------|------|
| 超音波 | 機械波，水傳得很好 | 軟組織、孕婦、心臟 |
| MRI | NMR | 全身軟組織 |

電磁波（包括 WiFi）**完全不在這個賽道上**。

## 結論

| 想做的事 | WiFi CSI 行嗎 |
|---------|-------------|
| 看內臟、看骨頭、看腫瘤 | ❌ 物理上不可能 |
| 監測呼吸、心跳、姿態 | ✅ 這正是 RuView 做的 |
| 非接觸生命徵象 | ✅ 商業上已有應用 |
| 取代 MRI / 超音波 | ❌ 完全不是同一個量級 |

如果你對「**無線非接觸醫療感測**」有興趣，**RuView 這個方向沒錯**（呼吸、心跳、跌倒偵測等），但放棄「看穿身體成像」這個念頭。要看內部，超音波或 MRI 還是唯二解。

要分清楚你問的是哪條線。ESC 跟 ESP32 之間是 **3-pin 連接器**（訊號 + 紅 + 黑），意義各不同。

## ESC 訊號接頭：3 條線各自的意義

```
ESC 一端           ESP32 一端
─────────         ─────────
SIG (白/黃) ───→  各自獨立 GPIO
+   (紅)    ───→  ESC 內建的 5V BEC 輸出
GND (黑)    ───→  接地
```

## 哪條可以共線，哪條不能

| 線 | 4 顆 ESC 可以共用一條線到 ESP32 嗎 |
|----|--------------------------------|
| **SIG 訊號（白）** | ❌ **絕對不行** |
| **GND（黑）** | ✅ **可以，這叫共地** |
| **+5V（紅）** | ⚠️ **不接 ESP32** |

## 詳細說明

### ① SIG 訊號線 — 絕對不能共線
每顆 ESC 需要**獨立的 PWM 訊號**才能各別控制四顆馬達轉速：
- M1 = 1500µs（中速）
- M2 = 1700µs（快）
- M3 = 1300µs（慢）
- M4 = 1500µs

如果 4 條 SIG 接同一條線：
- 4 顆馬達會收到完全相同的訊號 → **全部同速** → 飛機不能轉向、不能翻滾
- 4 顆 ESP32 GPIO 同時驅動一條線會打架（甚至燒 GPIO）

→ **必須各自接到 GPIO25 / 26 / 27 / 14**

### ② GND（黑）— 可以也應該共地
4 顆 ESC 的 GND **可以接在一起**，再用**一條線**到 ESP32 GND。這叫「共地」，是必須的。

實務做法：
```
ESC1 GND ┐
ESC2 GND ├──→ 焊在一起 → 一條線 → ESP32 GND
ESC3 GND ├
ESC4 GND ┘
```

理由：所有零件必須有共同的 0V 參考點，PWM 才能正確解讀。

> ⚡ 進階：PDB 上通常已經有 GND 大銅環，你也可以**每顆 ESC 各自接到 PDB 的 GND**，然後用一條線從 PDB GND 接 ESP32 GND，結果一樣。

### ③ +5V（紅）— 不要接 ESP32
ESC 內建 BEC 會輸出 5V，原本是給遙控接收器供電用。但：

- **4 顆 BEC 並聯 = 電壓打架**（每顆 BEC 認定的 5V 可能差 0.1V）
- 你已經有 **PDB 的 5V BEC** 給 ESP32 供電
- 多接會造成回流 / 電壓不穩 / 燒晶片

→ **4 條紅線都用熱縮套包起來不要接**（或剪掉只留訊號 + GND 兩條）

## 正確接線示意

```
┌─────────┐
│  ESP32  │
│ GPIO25 ─┼──────────────── ESC1 SIG (白)
│ GPIO26 ─┼──────────────── ESC2 SIG (白)
│ GPIO27 ─┼──────────────── ESC3 SIG (白)
│ GPIO14 ─┼──────────────── ESC4 SIG (白)
│         │            ┌─── ESC1 GND (黑)
│         │            ├─── ESC2 GND (黑)
│ GND    ─┼────────────┼─── ESC3 GND (黑)
│         │            └─── ESC4 GND (黑)
└─────────┘

（4 條紅線 = ESC BEC 5V）→ 不接，包熱縮套
```

## 你的「正負極」可能指動力線

**ESC 還有另外兩條粗線**（從 ESC 出來連到 PDB / 電池）：
- 紅粗線（11.1V 動力進）
- 黑粗線（GND 動力進）

**這兩條絕對不要接 ESP32！** 是 11.1V × 30A 的大電流，會立刻燒掉 ESP32。

正確接法：
```
電池 ─→ PDB ─→ 11.1V × 4 路 ─→ 4 顆 ESC 的動力輸入
       │
       └─→ 5V BEC ─→ ESP32 VIN
```

| 線 | 4 顆 ESC 處理方式 |
|----|------------------|
| 訊號（白）| **各自接不同 GPIO**（25/26/27/14）|
| GND（黑）| **全部共地到 ESP32 GND** |
| +5V（紅）| **不接，包起來** |
| 動力 11V 紅/黑粗線 | **接 PDB，不接 ESP32** |

## 為什麼不用買額外的 5V→3.3V 模組

| 路徑 | 已經處理好的東西 |
|------|---------------|
| 11.1V → 5V | ESC 內建 BEC 或 PDB 內建 BEC |
| 5V → 3.3V | **ESP32 DevKit 板上的 AMS1117**（800mA） |
| 5V → 感測器 | 不需要，所有感測器都吃 3.3V 從 ESP32 取 |

ESC1 紅線 ─→ 擴充板 5V 母線 ─┬─→ ESP32 VIN
                            ├─→ ESP32-CAM 5V
                            └─→ GPS 5V

ESC2/3/4 紅線 → 全部包起來不接
```

- 簡單、穩定、不會打架
- 缺點：所有負載集中在 ESC1，BEC 壓力大
- 適用：負載總和 < 1.5A（你目前的配置完全 OK）

## 🅱️ 進階做法：4 顆 ESC BEC 並聯（**必須加二極體**）

如果你想用上 4 顆 BEC 分擔負載：

```
ESC1 紅 ─→ Schottky 二極體 ─┐
ESC2 紅 ─→ Schottky 二極體 ─┼─→ 5V 母線 ─→ 各模組
ESC3 紅 ─→ Schottky 二極體 ─┤
ESC4 紅 ─→ Schottky 二極體 ─┘
```

**為什麼必須加二極體（Diode-OR）**：

- 4 顆 BEC 輸出電壓會有微小差異（例如 4.95V、5.00V、5.02V、4.98V）
- 沒二極體 → 電壓最高那顆**狂灌電給其他三顆**（電流倒流，叫 backfeed）
- 結果：高壓那顆過熱燒掉、低壓那顆也被反灌損壞
- 加二極體後：電流只能正向流出，倒流被擋住

**建議二極體**：1N5822（3A Schottky）或 SS34，正向壓降 0.3V 左右。

⚠️ **代價**：經過二極體後 5V 會掉到約 4.7V，ESP32 還是吃得下，但邊緣感測器可能會勉強。

## 🅲 危險做法：直接 4 顆 BEC 紅線焊一起

```
ESC1/2/3/4 紅線全部焊在同一焊點 ─→ ESP32 VIN
```

**絕對不要這樣做**：
- 第一秒可能沒事
- 第十秒高壓 BEC 開始狂灌低壓 BEC
- 第幾分鐘可能就燒掉了
- 即使沒燒，飛行中震動讓某顆 ESC 重啟瞬間電壓崩潰 → 整套系統當機 → **炸機**

## 🅳 最簡單做法：直接用 PDB 的 5V BEC

你買的電源分配板（PDB）大多**內建 5V BEC（5V/2~3A 切換式穩壓器）**，就是為了取代上述麻煩：

```
電池 ─→ PDB ─→ 11.1V × 4 ─→ ESC 動力
              │
              └─→ 5V BEC 輸出 ─→ ESP32 / 模組
```

- PDB 內建 BEC 通常**比 ESC BEC 強**（切換式效率 90%+，不像 ESC 線性式發熱）
- 不用焊任何東西
- **這是業界標準做法**

→ 你的元件清單裡 PDB 已經包含這個功能，**ESC 那 4 條紅線根本不用焊**，全部包起來就好。

## 關鍵差異：**被動源** vs **主動源**

| | 電池 | BEC（穩壓器）|
|---|------|------------|
| 種類 | **被動電壓源** | **主動穩壓電路** |
| 行為 | 「我就是這個電壓」 | 「我**強迫維持**這個電壓」 |
| 內阻 | 50~200mΩ（天然限流）| ~10mΩ 或更低 |
| 有回授迴路嗎 | ❌ 沒有 | ✅ 有，持續調整 |

## 電池並聯的物理：自然平衡

```
電池 A：4.20V，內阻 50mΩ
電池 B：4.10V，內阻 50mΩ
並聯瞬間：
   電流 I = (4.20-4.10) / (0.05+0.05) = 1A
   A 放電給 B，10 秒後兩邊變 4.15V → 達平衡
   電流歸零，安定下來 ✅
```

**自我穩定**：電壓高的會「自然」放電給低的，平衡後就沒事了。物理上**會找到平衡點**。

⚠️ **但有例外**：
- 兩顆電池電壓差**超過 0.5V** → 瞬間電流可能 5~10A → 可能燒接線 / 起火
- 不同型號 / 容量 / 新舊混用 → 持續循環不平衡 → 壽命減半
- 這就是為什麼 LiPo 並聯充電前要**先平衡每顆電壓接近**

## BEC 並聯的物理：互相對抗

```
BEC A：設定 5.02V（內部回授說「我要維持 5.02V！」）
BEC B：設定 4.98V（內部回授說「我要維持 4.98V！」）

並聯瞬間：
   A 看到輸出只有 5.00V（被 B 拉下來）
   → A 的回授說「不行！要拉到 5.02V」→ 加大電流
   B 看到輸出 5.00V（被 A 推上來）
   → B 的回授說「不行！要降到 4.98V」→ 想吸收電流
   
   A 拼命推、B 拼命吸 → 巨大循環電流 → 過熱
```

**不穩定**：兩邊都有「主動意志」，**永遠找不到平衡點**，互相對抗到燒掉。

## 用個比喻

| | 比喻 |
|---|------|
| **兩顆電池並聯** | 兩個水桶連通管 → 水位自然平衡 ✅ |
| **兩顆 BEC 並聯** | 兩個機器人各自抽水，A 想抽到 102 公分、B 想抽到 98 公分 → 兩個機器人**互相對抗到馬達燒掉** ❌ |

## 那為什麼飛機可以「同時用 4 顆 ESC」？

因為 4 顆 ESC 的**動力輸入端**（11.1V 那邊）是**並聯吃電池**，不是 BEC 互相並聯：

```
電池 11.1V ─┬─→ ESC1 → M1
            ├─→ ESC2 → M2     ← 這邊是「4 個負載並聯吃同一個電池」
            ├─→ ESC3 → M3        負載並聯沒問題（每個 ESC 各自獨立）
            └─→ ESC4 → M4

但是 BEC 輸出端：
ESC1 BEC ─┐
ESC2 BEC ─┤   ← 這邊才是「4 個電源並聯」
ESC3 BEC ─┤      電源並聯有打架問題
ESC4 BEC ─┘
```

**負載並聯**沒事，**電源並聯**會打架。

## 但其實 BEC 也可以並聯，只是要加東西

工業上**真的會把多個電源並聯**（伺服器、UPS 都是這樣冗餘）。做法：

| 方案 | 原理 |
|------|------|
| **Diode-OR** | 加二極體，只准電流正向流出 → 變成被動源 |
| **N+1 Active Sharing** | 高階電源有「電流分享接腳」（current share pin），透過數位通訊協調 |
| **Droop Sharing** | 故意讓電壓隨電流下降，造成自然分流 |

ESC 的 BEC 是**最廉價的線性穩壓**，沒這些機制 → 不能直接並聯。

## 一句話總結

| 並聯 | 自然平衡？ | 安全嗎 |
|------|---------|--------|
| **電池 / 電容** | ✅ 自我平衡 | 電壓差小就安全 |
| **沒有電流分享的 BEC** | ❌ 永遠對抗 | **會燒** |
| **有電流分享 / Diode-OR 的 BEC** | ✅ 可控 | 安全 |

> 你的觀察其實點出了**電子學的一個核心原理**：「電壓源並聯不能直接靠在一起」，**但被動源（電池）和主動源（穩壓器）的差別在於有沒有回授迴路**。

電池並聯不會炸，是因為它「沒有要堅持某個電壓」；BEC 並聯會炸，是因為它「拼命要堅持自己的電壓」。

## J9 I²S 麥克風預留說明

```
J9 接腳：  VCC ── 從板上 5V 取
           GND ── 從板上 GND 取
           WS  ── 你之後再飛線到 ESP32（I²S 字選）
           SCK ── 你之後再飛線到 ESP32（I²S 時脈）
           SD  ── 你之後再飛線到 ESP32（I²S 資料）
```
