## 1. 控制架構：Angle Mode（自穩）

```
  搖桿輸入                MPU6050
  Roll/Pitch 期望角度    實際角度 + 角速度
       │                       │
       └───────────┬───────────┘
                   ▼
              PID 計算
        ┌────┬─────┬─────┐
        │Roll│Pitch│ Yaw │
        └─┬──┴──┬──┴──┬──┘
          ▼     ▼     ▼
       ┌──────────────────┐
       │   馬達混控公式    │
       └────┬────┬────┬───┘
            ▼    ▼    ▼   ▼
           M1   M2   M3  M4
```

| 軸 | 控制模式 | 為什麼 |
|----|---------|-------|
| Roll / Pitch | **Angle PID**（角度誤差 → 修正量） | 搖桿放開飛機自動回水平 |
| Yaw | **Rate PID**（角速度誤差 → 修正量） | 沒磁力計，無法定絕對航向 |

## 2. 馬達混控公式（X 型配置）

依你的馬達定義：

```
M1 (前左 CW)  = T + pitchOut + rollOut - yawOut
M2 (前右 CCW) = T + pitchOut - rollOut + yawOut
M3 (後左 CCW) = T - pitchOut + rollOut + yawOut
M4 (後右 CW)  = T - pitchOut - rollOut - yawOut
```

直覺理解：
- **要往前飛** → 後方馬達加速、前方減速 → pitch 增大
- **要向右滾** → 左方馬達加速、右方減速 → roll 增大
- **要逆時針 yaw** → CW 馬達加速、CCW 減速 → 反扭矩失衡

## 3. PID 實作關鍵細節

**單軸 PID 公式：**
```
error      = setpoint - measurement
P_term     = Kp * error
I_term    += Ki * error * dt          ← 累加，並 clamp 防飽和
D_term     = -Kd * gyro_rate          ← 用陀螺儀代替誤差微分
output     = P_term + I_term + D_term
```

**三個防呆機制（沒做飛機會炸）：**

| 機制 | 為什麼 | 怎麼做 |
|------|-------|--------|
| Anti-windup（防積分飽和）| 飛機被卡住時 I 項會無限累積 | I_term 做 `constrain(-100, 100)` |
| Derivative on Measurement（D 用量測而非誤差）| 搖桿突變時誤差微分會爆衝 | 直接用 `gyro_rate`（已是角速度）當 D |
| Throttle-gated PID | 地面靜止時不要嘗試穩定 | 油門 < 10% 時 I 項歸零、輸出為 0 |

## 4. 軟體結構（加在現有程式哪裡）

只動 `loop()` 加一個函式：

```cpp
void loop() {
  recvData();
  failsafe();
  checkArm();
```

`pidControl()` 內部：
```cpp
// 把 TX 搖桿（0~255）轉成期望角度（±30°）
float rollSet  = (data.roll  - 127) * (30.0 / 127.0);
float pitchSet = (data.pitch - 127) * (30.0 / 127.0);
float yawSet   = (data.yaw   - 127) * (90.0 / 127.0);  // 期望角速度

// 三軸獨立 PID
rollOut  = pidRoll.compute(rollSet,  roll,    gyroX);

```

姿態讀取在 `readAttitude()` 函式裡，核心是**互補濾波（Complementary Filter）**。我拆三步說。

## 步驟 1：從 MPU6050 拿原始資料

```cpp
int16_t ax, ay, az, gx, gy, gz;
mpu.getMotion6(&ax, &ay, &az, &gx, &gy, &gz);
```

兩種感測器各拿三軸：

| 感測器 | 輸出 | 物理意義 |
|--------|------|---------|
| 加速度計 | ax, ay, az | 三軸加速度（含重力，靜止時讀到 1g 向下）|
| 陀螺儀 | gx, gy, gz | 三軸**角速度**（°/s）|

## 步驟 2：兩種方法各自算姿態

### 方法 A：加速度計 → 靜態傾角

當飛機**靜止**時，加速度計只讀到重力向量。重力指地心，所以可以推算機身傾角：

```cpp
float accRoll  = atan2f(ay, az) * 57.2958f;
float accPitch = atan2f(-ax, sqrtf(ay*ay + az*az)) * 57.2958f;
```

幾何上：
```
       重力 g
         │
         ▼
    ┌────────┐
    │        │  ← 機身傾斜 θ
    └────────┘
       az = g·cos(θ)
       ay = g·sin(θ)
       ∴ θ = atan2(ay, az)
```

**優點**：絕對參考，不會漂移
**缺點**：飛機加速、震動時讀值會亂

### 方法 B：陀螺儀 → 角速度積分

陀螺儀直接給角速度，乘上時間就是角度變化：

```cpp
float gRoll = gx / 131.0f;   // 原始 int16 → °/s
roll += gRoll * dt;          // 角度 = 角速度 × 時間
```

131 是 MPU6050 在 ±250°/s 範圍下的 LSB 換算係數。

**優點**：快速、不受震動影響
**缺點**：**會漂移**（偏移誤差持續累積，10 秒後可能差 5°）

## 步驟 3：互補濾波融合

把兩個方法各取所長：

```cpp
roll = 0.98f * (roll + gRoll * dt) + 0.02f * accRoll;
```

| 項 | 來源 | 作用 |
|----|------|------|
| `0.98 × (前一刻 + 陀螺儀積分)` | 陀螺儀路徑 | **短期快速追蹤** |
| `0.02 × accRoll` | 加速度計路徑 | **長期校正漂移** |

**直覺**：每一拍我相信 98% 的陀螺儀（因為它反應快），但留 2% 用加速度計「拉回真實垂直方向」。長期下來陀螺儀的累積漂移會被加速度計持續修正。

效果類比：
- 純陀螺儀：開高速攝影機，影像清楚但畫面會慢慢飄走
- 純加速度計：每秒拿羅盤對齊，但有人撞桌子就亂跳
- 互補濾波：兩個一起用，互相補

## Yaw 軸特殊處理

```cpp
yawRate = gz / 131.0f;   // 只存角速度，不積分
```

**沒有處理絕對 yaw 角度**，原因：
- 加速度計**看不到 yaw**（繞垂直軸轉，重力向量不變）
- MPU6050 **沒有磁力計**（這顆要 MPU9250 才有）
- 沒有絕對參考 → 無論怎麼積分都會漂移

所以 PID 對 yaw 只控制「**轉動速度**」而不是「**指向角度**」。搖桿打 yaw → 飛機轉那麼快、放開停。

## 完整時序圖

```
時間
 │ MPU 取樣 (1kHz)
 │     │ 加速度 → atan2 → accRoll/accPitch
 │     │ 陀螺儀 / 131  → gRoll/gPitch/yawRate
 │     │
 │     ▼
 │  互補濾波
 │   roll  ← 0.98 × (roll  + gRoll  × dt) + 0.02 × accRoll
 │   pitch ← 0.98 × (pitch + gPitch × dt) + 0.02 × accPitch
 │   yawRate ← gz/131  (不存歷史)
 │     │
 │     ▼
 │  PID 控制器使用
 ▼
```

## 這個方法的限制

| 場景 | 表現 |
|------|------|
| 室內懸停 | ✅ 很好 |
| 慢速移動 | ✅ 很好 |
| 急加速 / 大幅度動作 | ⚠️ 加速度計讀到非重力 → 暫時誤判 |
| 機身震動大（馬達不平衡）| ⚠️ accel 雜訊大 → 角度抖動 |
| 飛 10 分鐘以上 | yaw 會慢慢偏 → 自動飛行時要注意 |
