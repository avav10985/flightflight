有，**寫死了**。在三個程式裡都這樣定義：

```cpp
#define PIN_ESC1     25   // M1 前左 CW
#define PIN_ESC2     26   // M2 前右 CCW
#define PIN_ESC3     27   // M3 後左 CCW
#define PIN_ESC4     14   // M4 後右 CW
```

## 對應關係

```
俯視（USB 朝下時看 LOLIN）：

            前方
                
   M1 ◯─────◯ M2     M1 = GPIO 25, 前左, CW
   GPIO 25    GPIO 26  M2 = GPIO 26, 前右, CCW
              
              
   M3 ◯─────◯ M4     M3 = GPIO 27, 後左, CCW
   GPIO 27    GPIO 14  M4 = GPIO 14, 後右, CW

            後方
```

## 為什麼這個對應**很重要**

PID 階段的 X 型混控公式**寫死配對 M1~M4 的位置**：

```cpp
int m1 = baseT + pitchOut + rollOut - yawOut;   // 前左 CW
int m2 = baseT + pitchOut - rollOut + yawOut;   // 前右 CCW
int m3 = baseT - pitchOut + rollOut + yawOut;   // 後左 CCW
int m4 = baseT - pitchOut - rollOut - yawOut;   // 後右 CW
```

如果 GPIO 25 上面的馬達**裝在飛機的後右**（不是前左），PID 計算「前傾」時會去**加錯邊**，結果飛機**反向翻覆**。

## 你要做的事

接線時**物理上確認**：

| 程式裡的 GPIO | 必須接到哪個位置的馬達 |
|------------|-------------------|
| GPIO 25 | **前左**（從上俯視時的左前角）|
| GPIO 26 | **前右** |
| GPIO 27 | **後左** |
| GPIO 14 | **後右** |

**簡單檢查方式：**
1. 燒 [Drone_FC_Manual.ino](Drone_FC_Manual/Drone_FC_Manual.ino)
2. ARM、推一點點油門
3. **暫時把 ESC2/3/4 訊號線從 LOLIN 拔掉**，只留 ESC1（GPIO 25）
4. **前左** 那顆馬達應該轉，其他三顆不轉
5. 對不上 → 把 LOLIN 上的 ESC1 訊號線跟其他互換

## 如果你硬要保留現在的接線

也可以直接改程式 `#define`：

```cpp
// 假設你實際接線是：
//   GPIO 25 → 後右
//   GPIO 26 → 後左
//   GPIO 27 → 前右
//   GPIO 14 → 前左

// 程式改成：
#define PIN_ESC1     14   // M1 前左 (改腳位)
#define PIN_ESC2     27   // M2 前右
#define PIN_ESC3     26   // M3 後左
#define PIN_ESC4     25   // M4 後右
```

PID 混控公式不變，只是把「GPIO 編號」對應到正確的馬達。

## 順帶：馬達**旋轉方向**也寫死

PID 公式裡 yaw 的正負號也假設：
- M1 (前左) + M4 (後右) = **CW**
- M2 (前右) + M3 (後左) = **CCW**

裝螺旋槳前必須確認**馬達轉向**符合這個（用手撥馬達看方向）。轉錯的就把 ESC 跟馬達之間的 3 條三相線**任 2 條對調**。

ESC「逼逼叫」通常是它**找不到有效訊號**或**沒被武裝**。先確認三件事：

## ① 電池有沒有接到 ESC？

ESC 需要**電池供電**才能工作。USB 只給 ESP32 電，**不能驅動馬達**。

```
電池 → PDB → ESC 的「粗紅黑線」  ← 這要接
USB  → LOLIN D32                ← 你已接

ESC 訊號線（白）→ LOLIN GPIO    ← 你已接
ESC GND（黑）→ LOLIN GND       ← ★ 這條接了嗎？
```

**沒接電池**：ESC 完全沒電，連叫都叫不出來。聽得到逼逼聲表示有電。

**接電池後叫不停**：訊號問題（見 ②）。

## ② ESC 黑線（GND）有接 LOLIN 嗎？

「只接訊號線」聽起來可能漏了 GND！

```
ESC 訊號線（白）─→ GPIO 25 ✓
ESC GND（黑）  ─→ LOLIN GND ★ 必須接
ESC 紅線（5V）─→ 不接（包熱縮套）
```

**沒共地** → LOLIN 的 PWM 訊號浮接 → ESC 看不懂 → **逼逼叫**

## ③ 燒對程式了嗎？

如果你燒的是 `Drone_FC_PID.ino`，**ENABLE_NRF24 是強制開的**（沒有旗標可關）。如果 NRF24 沒接 → 程式可能停在 `radio.begin()` → ESP32 沒送 PWM → ESC 收不到 → 逼逼叫

**對策**：先燒 [Drone_FC_Manual.ino](Drone_FC_Manual/Drone_FC_Manual.ino) 或 [Drone_FC_Gyro.ino](Drone_FC_Gyro/Drone_FC_Gyro.ino) 看序列埠有沒有開機訊息。

## 診斷流程

**Step 1**：序列埠 (115200) 有沒有看到 `[3] ESC 初始化 1000µs`？
- 沒看到 → 程式卡住或燒錄問題
- 看到 → ESP32 有在送 1000µs 信號

**Step 2**：用三用電錶量 LOLIN GND ↔ ESC 黑線
- 通 = 共地 OK
- 不通 = ★ 元兇

**Step 3**：仔細聽逼逼聲的**節奏**

| 聲音 | 意義 | 解法 |
|------|------|------|
| 持續快速「嗶嗶嗶嗶」 | 沒收到訊號 | 接 GND、確認訊號線 |
| 慢慢「嗶 ── 嗶 ── 嗶」 | 等待校準 | 程式有送 1000µs，等幾秒 |
| 開機「嗶嗶嗶ーー嗶ーー嗶」後安靜 | **正常**！表示識別電池節數 | 沒事 |
| 「嗶嗶嗶 嗶嗶嗶」連續一直響 | 油門太高（不在 1000µs） | 程式問題或燒錯 |

**要分兩處裝**，目的不同：

## ① 擴充板上的 C1 + C2 → **5V 軌**

```
ESC1 BEC 5V ─┬─→ ESP32 USB 腳
             ├─→ ESP32-CAM 5V
             ├─→ GPS 5V
             │
            C1 (100µF)
            C2 (0.1µF)
             │
            GND
```

**用途：**
- 吸收 **ESC BEC 的雜訊**（馬達切換時 5V 會抖動）
- 馬達啟動瞬間電壓掉時，靠 C1 補電給 ESP32 / CAM / GPS
- 多顆模組共用 5V 時，互相影響也被吸收

裝在 **5V 軌與 GND 之間，並聯**，越靠近**電源進來那一端**（ESC1 紅線焊點）越好。

## ② NRF24 旁邊的 100µF → **3.3V 軌**

NRF24 有**獨立的去耦電容**，**焊在 NRF24 模組腳座旁邊**，不在擴充板上：

```
NRF24 VCC (3.3V) ─┬─→ NRF24 模組 VCC
                  │
              C3 (100µF) ← 必須在這裡，越近 NRF24 越好
                  │
NRF24 GND ────────┴── 共地
```

**為什麼必須單獨在 3.3V 上：**
- NRF24+PA/LNA **發射瞬間電流可達 115mA**
- LOLIN 的 AMS1117 3.3V 反應不夠快 → **電壓瞬間崩潰** → NRF24 重置
- 100µF 在旁邊當「電池」，瞬間放電撐過去
- **沒這顆 NRF24 會反覆斷線**

## 配置摘要

| 電容 | 位置 | 軌 | 距離 |
|------|------|----|------|
| C1 100µF | 擴充板 | 5V | 近 ESC1 進電端 |
| C2 0.1µF | 擴充板 | 5V | 跟 C1 並聯 |
| **C3 100µF** | **NRF24 模組腳座** | **3.3V** | **越近越好（< 1cm）** |

## 如果你只買一組

優先順序：
1. **NRF24 旁邊那顆 100µF** ← 最重要，不裝直接 NRF24 不穩
2. 擴充板 5V 的 C1 (100µF)
3. 擴充板 5V 的 C2 (0.1µF) ← 高頻濾波，最後加

## 加裝陶瓷電容（104）位置進階

如果還有多的 0.1µF（104）陶瓷：
- 放在每個模組 VCC ↔ GND 附近：MPU6050、ESP32-CAM、GPS 各 1 顆
- 每顆模組就近濾高頻噪音，效果疊加
