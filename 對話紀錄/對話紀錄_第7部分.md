## 怎麼做（架構）

```
無人機端：
  ESP32 + 4G LTE 模組（SIM7600 / EC200S / A7670）
  + 鏡頭（ESP32-CAM 或單獨 CSI 相機）
        ↓ 影像走 4G
網際網路
        ↓
雲端 / 自架 RTMP / WebRTC 伺服器
        ↓ 串流分發
地面：手機 App / 電腦瀏覽器
```

## 硬體選項

| 模組 | 規格 | 價格 | 備註 |
|------|------|-----|------|
| **SIM7600A / SIM7600E** | 4G LTE Cat 4，~10Mbps | NT$1500~2500 | 老牌穩定，需 SIM 卡 |
| **EC25** | 4G LTE Cat 4 | NT$2000 | Quectel 品牌 |
| **A7670** | 4G LTE Cat 1 | NT$1000 | 較便宜 |
| **SIM7600 + ESP32-CAM 整合板** | 一體式 | NT$3000+ | 比較貴但省事 |

加上：
- **物聯網 SIM 卡**（中華 / 台哥 IoT 卡，月費 NT$200~400）
- 或一般 4G 上網卡（吃流量比較貴）

## 跟 5.8G FPV 比較

| 項目 | 5.8G 類比 FPV | 4G LTE |
|------|--------------|--------|
| 延遲 | **<40ms** | **200~500ms** |
| 射程 | 1~5km | **任何有訊號的地方**（理論無限）|
| 畫質 | 480p 模糊 | 720p~1080p |
| 抗遮蔽 | 直線需見 | 可穿牆繞屋 |
| 多人觀看 | 1 對 1 | **多人可同時看**（雲端分發）|
| 耗電 | 0.5W | 1.5~3W（高） |
| 法規 | 25mW 合法、>25mW 灰色 | **疑慮（飛行中用 4G 多國禁止）**|
| 設備成本 | NT$1500 | NT$2500+ + 月費 |

## 致命缺點：延遲

4G 走網際網路 → 經過手機基地台、骨幹、雲端伺服器，**單程 100~250ms**。

**這個延遲不能做 FPV 操控**（你看到障礙物時已經撞上去）。只能做：
- 巡檢拍攝（不靠畫面操控）
- 直播紀錄
- 物體追蹤（電腦端 AI 慢慢分析）

## 法規（重要）

**台灣 NCC 規定：** 飛行中的航空器（無人機）**使用蜂巢式行動通訊原則上不允許**，因為基地台訊號設計給地面，空中用會造成基地台亂跳、影響通訊品質。**實務上常見繞道**（用 IoT 物聯網卡、低空），但出事是你的責任。

**其他國家**（美 FCC、歐 ETSI、中國）也多有限制。

## 推薦用 LCD 2004A

**好消息**：你的這個 2004A 已經**焊好 I²C 轉接板（PCF8574T）**！正面 16 隻針都被轉成右上角 4 隻針：

```
GND
VCC （5V）
SDA → Nano A4
SCL → Nano A5
```

**只要 4 條線**就能接 Nano，跟我之前說的 SSD1306 OLED 一樣簡單，**而且字大很多**（每個字元 1cm，老花眼也看得清）。

## 顯示版面（20×4 = 80 字夠用）

```
┌────────────────────┐
│ARM Bat:11.8V Sat:8 │   ← 武裝、電池、GPS 衛星
│R:-2.1 P:3.7 Y: 45  │   ← 姿態角
│Alt:1.2m   F:1500mm │   ← 高度 + 前方避障距離
│Thr:80% Mode:STAB   │   ← 油門 + 飛行模式
└────────────────────┘
```

比 OLED 0.96" 多 4 倍可顯示字數。

## 函式庫

工具 → 管理函式庫 → 搜「**LiquidCrystal_I2C**」by Frank de Brabander 或 Marco Schwartz。

範例程式碼：
```cpp
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

LiquidCrystal_I2C lcd(0x27, 20, 4);  // 地址通常 0x27 或 0x3F

void setup() {
  lcd.init();
  lcd.backlight();
  lcd.setCursor(0, 0);
  lcd.print("ARM Bat:11.8V Sat:8");
}
```

| 觀察 | 判定 |

## 為什麼高度是 91 m

公式：高度 = 氣壓 × 公式推回去（假設海平面 1013.25 hPa）

```
你的氣壓 1002.27 hPa → 推算 91.8 m
```

**這是「相對海平面」的高度**。實際你可能在 5m 樓上，但今天氣壓比標準低 11 hPa，所以推算結果偏高 86m。

> **無人機飛行時不在乎絕對值**，只看「**相對開機時的高度差**」，所以你的飛控程式裡有 `baseAltitude = 開機高度當基準`，飛行時的高度 = `當下 - baseAltitude`。

## 動態測試（確認真的會反應）

試三件事：

1. **抬高感測器 30cm** → 高度應該變小（氣壓上升=空氣多=海拔降，等等錯了）

   實際上：**抬高 = 高度 +0.3m 左右**（你在更高的地方了）

2. **嘴對著感測器輕吹**（暫時提高局部氣壓）→ 高度數值會跳

3. **手指捂住感測器幾秒**（升溫）→ 溫度應該上升 1~2°C

任一個有反應 → 晶片活的、能用。

## 怎麼看

| 欄位 | 意義 |
|------|------|
| **用於定位** | 真正參與計算位置的衛星（越多越準）|
| **可見** | 天空中能收到訊號的所有衛星 |
| 進度條 `[#######....  ]` | `#` = 用於定位、`.` = 可見但未用、空白 = 沒收到 |
| `[#03:42dB]` | 衛星 PRN 編號 3、訊號強度 42 dB |
| **HDOP** | 水平精度，越小越好（< 2 = 好、< 5 = 可用）|

## 訊號強度 dB 參考

| SNR | 品質 |
|-----|------|
| > 40 dB | 極好 |
| 30~40 dB | 良好 |
| 20~30 dB | 普通 |
| < 20 dB | 弱（不可靠）|

通常看到 **4 顆以上 > 30 dB**，就能取得穩定 fix。

## 為什麼有時候 PRN 顯示但 SNR 空

PRN 出現代表「天空中存在這顆衛星」，但訊號太弱沒辦法接收，所以 SNR 為 0（顯示 `--`）。**搬到開闊處或等更久**，弱訊號的衛星會逐漸鎖定。

## 怎麼用手機看

2. 左上角選單 → Devices → Bluetooth Classic → 點 `GPS_TEST_D32`

## 戶外攜帶配置

```
行動電源 ──USB線──→ LOLIN D32 USB 孔
                       │
                       ├── MPU6050（如果還接著）
                       └── GPS NEO-6M（重點，天線朝天）

手機 ──藍牙(10m內)──→ ESP32-CAM
```

**只要：行動電源、ESP32、GPS、手機**（不用筆電）。

## 注意

- 階段 1 的**原始 NMEA 資料**只在 USB 端印（藍牙吞不下太多原始資料會慢）
- 階段 2 結構化狀態（衛星數、SNR、座標）**藍牙照樣收到**
- 藍牙範圍 10m 內，所以你要**站在 ESP32 旁邊**看，但**人不用扛筆電**
