## 為什麼現在只能讀到 2 種

你的三段開關**物理上**有 3 個位置，但**電氣上**通常是這樣接的：

```
3 段開關（ON-OFF-ON 型）：
    上位 ──┐
    中位 ──┼── 共腳（COM）
    下位 ──┘
    
    撥到「上位」：COM 接通「上位腳」
    撥到「中位」：COM **誰都不通**（懸空）
    撥到「下位」：COM 接通「下位腳」
```

你目前的 [tx.JPG](tx.JPG) **只接了一個位置（上或下）+ COM**，所以：

| 開關位置 | 那條被接的腳 | digitalRead |
|---------|------------|------------|
| 上位（接通） | LOW（對 GND） | 0 |
| 中位（懸空） | HIGH（INPUT_PULLUP 拉高） | 1 |
| 下位（**沒接線**）| HIGH（拉高）| 1 |

中位和下位讀起來一樣 → **只能分 2 種**。

## 改成 3 段：每個開關多接一條線

```
3 段開關（ON-OFF-ON）：

     上位腳 ─────→ D0 (現在已接)
     共腳   ─────→ GND
     下位腳 ─────→ D2 (新增！多一條線)
```

軟體讀兩個腳的組合：

| 開關位置 | D0 | D2 | 判讀 |
|---------|-----|-----|------|
| 上位 | **LOW** | HIGH | 上 (aux=2) |
| 中位 | HIGH | HIGH | 中 (aux=1) |
| 下位 | HIGH | **LOW** | 下 (aux=0) |

兩個 GPIO **不可能同時 LOW**（因為共腳只接一邊），所以三種狀態獨立可辨。

## 程式怎麼改

```cpp
// 多一個腳位 pinMode
pinMode(0, INPUT_PULLUP);   // SW1 上位
pinMode(2, INPUT_PULLUP);   // SW1 下位（新增）
pinMode(3, INPUT_PULLUP);   // SW2 上位
pinMode(4, INPUT_PULLUP);   // SW2 下位（新增）

// 讀取邏輯
byte readSwitch(int pinUp, int pinDown) {
  if (digitalRead(pinUp) == LOW) return 2;     // 上位
  if (digitalRead(pinDown) == LOW) return 0;   // 下位
  return 1;                                     // 中位
}

void loop() {
  // ...
  data.aux1 = readSwitch(0, 2);   // 0 / 1 / 2 三種值
  data.aux2 = readSwitch(3, 4);
}
```

## 接線總表

### ① ESP32 ↔ MPU6050（4 條線）

| MPU6050 腳 | 顏色建議 | ESP32 腳 |
|-----------|--------|---------|
| VCC | 紅 | **3V3** |

| SDA | 綠 | **GPIO 21** |
| SCL | 黃 | **GPIO 22** |

### ② ESP32 ↔ ESC × 4（訊號 + 共地）

每顆 ESC 接出來是 **3-pin 接頭**（白/紅/黑）：

| ESC | 白線（訊號）→ ESP32 | 紅線（5V BEC）| 黑線（GND）→ ESP32 |
|-----|------------------|--------------|-------------------|
| **ESC1** 前左 | **GPIO 25** | **接 VIN**（給 ESP32 供電）| **GND** |

### ③ 動力線（粗紅黑線）

ESC 還有兩條**粗紅+粗黑**（從 ESC 另一端出來）：

4 顆 ESC 粗紅黑 → 全部接 PDB（電源分配板）

```

3. 開序列埠 (115200) 看到 "MPU6050: OK"
4. 搖晃飛機，R/P 角度跟著變
```

**Step 3：接 BEC 取電（拔 USB）**
```
1. ESC1 紅線焊上 ESP32 VIN
2. 接電池
3. 看 ESP32 紅色 PWR LED 亮
4. 用 USB 接電腦看序列埠（USB + BEC 同時通電沒問題，板上 diode 處理）

**Step 4：測 ESC**
```
1. 4 顆 ESC 動力線接 PDB（粗紅黑）

3. 通電聽 ESC 嗶聲（識別最低油門）

## I²C 是「共用匯流排」

**一條 SDA + 一條 SCL 可以掛很多個 I²C 元件**，只要每個的地址不同。地址撞了才需要多工器。

## 你的元件 I²C 地址

| 元件 | I²C 地址 | 需要 TCA9548A 嗎 |
|------|---------|----------------|
| MPU6050 | 0x68 | ❌ 直接接 |
| BMP280 | 0x76 | ❌ 直接接 |
| TCA9548A 本身 | 0x70 | — |
| VL53L0X #1 | 0x29 | ✅ 要 |
| VL53L0X #2 | 0x29 ← **撞了** | ✅ 要 |

VL53L0X 沒辦法改地址（出廠就 0x29），所以兩顆掛同一條 I²C 線會打架 → **這時才需要 TCA9548A 切換** → 主控說「我現在跟 CH0 講話」 → 只有 CH0 的 VL53L0X 回應，其他被隔開。

## 你的元件可以分這幾類

| 元件 | 介面 | 接到 ESP32 哪些腳 |
|------|------|------------------|
| **MPU6050** | I²C | GPIO 21 SDA、22 SCL |
| **BMP280** | I²C | 同上（共用 SDA/SCL）|
| **TCA9548A → VL53L0X×N** | I²C（透過多工器）| 同上（共用 SDA/SCL）|
| **NRF24L01+PA/LNA** | **SPI** | GPIO 4、5、18、19、23 |
| **GPS NEO-6M** | **UART** | GPIO 16、17 |
| **ESC × 4** | **PWM** | GPIO 25、26、27、14 |
| 電池監測 | **ADC** | GPIO 34 |
| ESP32-CAM | **獨立 WiFi** | 不接 ESP32 |

**只有前三項是 I²C**。

## 接線示意（I²C 共用主幹）

```
ESP32 SDA (GPIO 21) ─┬───────┬───────┬─────── ...（可以再掛）
                     │       │       │
ESP32 SCL (GPIO 22) ─┼─┬─────┼─┬─────┼─┬───── ...
                     │ │     │ │     │ │
                  ┌──┴─┴──┐ ┌─┴─┴──┐ ┌┴─┴────┐
                  │MPU6050│ │BMP280│ │TCA9548A│
                  │ 0x68  │ │ 0x76 │ │ 0x70   │
                  └───────┘ └──────┘ └───┬────┘
                                         │
                                  ┌──────┼──────┐
                              CH0 │  CH1 │  CH2 │ ...
                                  │      │      │
                              VL53#1 VL53#2 (...)
                              (0x29) (0x29)
```

**MPU6050 / BMP280 直接掛在主幹**，**只有 VL53L0X 透過 TCA9548A 切換**。

## 給擴充板的意義

擴充板上 I²C 區段這樣設計：

```
┌─────────────────────────────────┐
│   I²C 主幹（SDA + SCL）         │
│   ↓ 從 LOLIN GPIO 21、22 進來   │
├─────────────────────────────────┤
│ ┌────┐ ┌────┐ ┌────┐ ┌────┐    │
│ │MPU │ │BMP │ │TCA │ │擴充│    │
│ │6050│ │280 │ │9548│ │用  │    │ ← 全部 4-pin
│ │ ✕4 │ │ ✕4 │ │ ✕6 │ │ ✕4 │    │   (VCC GND SDA SCL)
│ └────┘ └────┘ └────┘ └────┘    │
└─────────────────────────────────┘
（VL53L0X 接 TCA9548A 那組 6-pin）
```

**所有 I²C header 的 SDA、SCL 都並聯到主幹**，是「分支」不是「串接」。
