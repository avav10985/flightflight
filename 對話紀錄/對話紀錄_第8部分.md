## ① + ② 30kΩ + 10kΩ → **電池電壓監測**

```

      ├──→ LOLIN GPIO 34 (ADC)
      │
     10kΩ (R2)
      │
     GND
```

**為什麼要分壓**：
- ESP32 ADC 最大只能讀 **3.3V**
- 但 3S 電池滿電 **12.6V** → 直接接會燒 GPIO
- 分壓比 = 10/(30+10) = **1/4**
- 12.6V × 1/4 = 3.15V ✓（在 ADC 範圍內）

**用途**：程式讀 GPIO34 的電壓 × 4 = 真實電池電壓 → 低電量警告 / 自動返航。

## ③ 1kΩ → **電源指示 LED 限流**

```
5V ──→ 1kΩ (R3) ──→ LED ──→ GND
```

**為什麼要限流**：
- LED 直接接 5V 會**燒掉**（電流太大）
- 1kΩ 把電流限制到 `(5V - 2V) / 1000Ω ≈ 3mA`
- LED 正常亮、不燒

**用途**：擴充板上電後 LED 亮，**一眼確認有沒有通電**。

| 30kΩ | 1 | 電池分壓（上）| 擴充板 / 飛控 |

**沒衝突**，這兩件事是分開的：

- **ESP32 的「電源」** → ESC BEC 給的 5V
- **電池監測的「量測點」** → 接到**原始電池 11.1V**（不是 5V）

## 怎麼接

```
電池 11.1V ──┬──→ PDB ──→ ESC 動力 ──→ BEC 5V ──→ ESP32 供電
             │
             └──→ 30kΩ ──┬──→ GPIO34（只是「量電壓」，不是供電）
                          │
                         10kΩ
                          │
                         GND（共地）
```

**重點**：
- 分壓電路的**上端**接到「**電池正極 / PDB 的 11.1V**」那條
- ESP32 的 **GPIO34 只是「感測」這個電壓**，不從這裡取電
- ESP32 該怎麼供電還是照舊（BEC 5V）

## 為什麼可以這樣

ADC 是「**電壓表**」，只是「**看**」電壓多少，幾乎不吃電流（GPIO34 內阻極高）。

就像你用三用電錶量電池電壓 ——
- 電錶不是靠那兩根探針供電
- 探針只是「碰一下看數字」

GPIO34 + 分壓電阻就是這個「探針」，碰在電池 11.1V 上看數字。

## 量測點哪裡找 11.1V

| 位置 | 有沒有 11.1V |
|------|------------|
| 電池正極 | ✅ |
| PDB 的電池輸入端 | ✅ |
| XT60 接頭 | ✅ |
| ESC 動力線（粗紅）| ✅ |
| **BEC 5V 輸出** | ❌ 只有 5V，量不到電池真實電壓 |

分壓電路上端接**任何有 11.1V 的點**都行，最方便是 PDB。

## 為什麼不直接量 5V BEC

因為 **BEC 是穩壓的**：
- 電池從 12.6V 掉到 9V，BEC 輸出**永遠是 5V**（穩壓器的工作）
- 量 5V → 永遠看到 5V → **看不出電池快沒電**

所以一定要量「**沒穩壓的原始電池電壓**」才知道剩多少電。

## 共地是關鍵

分壓電路的下端 GND，必須跟 ESP32 GND **連通**（你的系統已經共地了）。否則 ADC 沒有共同參考點，讀值無意義。

## 結論

| 項目 | 接哪 |
|------|------|
| ESP32 供電 | ESC BEC 5V → USB 腳（不變）|
| 電池監測上端 | **電池 11.1V**（PDB / XT60）|
| 電池監測下端 | GND（共地）|
| 量測訊號 | 分壓中點 → GPIO34 |

## 功率計算

分壓電路：30kΩ + 10kΩ 串聯，跨在 12.6V（3S 滿電）：

**電流**：
```
I = V / (R1 + R2) = 12.6V / (30000 + 10000)Ω
  = 12.6 / 40000
  = 0.315 mA   ← 超小
```

**每顆電阻的功率**：
```
30kΩ：P = I²R = (0.000315)² × 30000 ≈ 3 mW
10kΩ：P = I²R = (0.000315)² × 10000 ≈ 1 mW
```

## 跟電阻額定比

| | 數值 |
|--|------|
| 你的電阻實際消耗 | **~3 mW** |
| 一般電阻額定 | **250 mW（1/4W）** |
| 用到幾分之幾 | **約 1/80** |

**只用到額定的 1.2%**，電阻**摸起來完全不會熱**。

## 為什麼這麼安全

關鍵在**電阻值很大**：

```
電阻大 → 電流小 → 發熱小（P = I²R）
```

- 30k + 10k = 40kΩ 很大 → 電流只有 0.3mA
- 電流小 → 即使電阻大，功率還是極小

這就是為什麼**量電壓用的分壓電阻故意選大阻值**：
1. 電流小 → **不浪費電池電量**（0.3mA 對 2200mAh 電池微不足道）
2. 發熱小 → **不會燒**

## 反例：什麼時候會燒

如果你用**小電阻**，例如 100Ω + 33Ω：
```
I = 12.6 / 133 = 95 mA
100Ω 功率 = 0.095² × 100 = 0.9 W   ← 超過 1/4W 額定 → 燒掉冒煙
```

所以**分壓電阻一定要用 kΩ 等級**（你的 30k/10k 完美）。

## LED 限流電阻會燒嗎

```
P = 0.003² × 1000 = 9 mW
```

|------|---------|------|--------|
| 30kΩ | 3 mW | 250 mW | ✅ 超安全 |

**好問題，這是飛控的核心觀念。** 你的直覺對：每次放置略有不同，但**加速度計會解決這個問題**。

## 關鍵：加速度計永遠知道「地心方向」

```
加速度計 = 重力感測器
不管你怎麼放，它永遠測得到「重力往哪指」= 真正的「下」
```

所以「水平」不是靠你**放得正不正**來定義，而是靠**重力**。互補濾波會用加速度計把姿態**慢慢拉回真實水平**。

## 三種「校正」要分清楚

| 校正 | 校什麼 | 多久一次 | 你的飛機要靜止嗎 |
|------|--------|---------|----------------|
| **陀螺儀偏移** | gyro 的零點漂移 | **每次開機** | ✅ 要（3 秒別動）|
| **加速度計水平** | IMU 裝歪的角度 | **一次就好** | ✅ 放水平面校 |
| **飛行中自穩** | 即時姿態 | 持續 | ❌（飛行中） |

## 市售飛控怎麼做

### ① 陀螺儀偏移：每次開機校
- 開機那幾秒 gyro 必須靜止
- 這是為什麼 DJI / Betaflight 開機要「放著別動幾秒」
- 校的是 gyro 漂移，**不是水平**

### ② 加速度計水平：**只校一次，存起來**
- 第一次設定時放在**真正水平的桌面**校一次
- 記錄「IMU 裝歪了幾度」這個固定偏移
- **存到 flash**，之後每次開機直接用，不重校

### ③ 飛行時：加速度計自動找水平
- 飛機飛起來，加速度計持續告訴飛控「重力在哪」
- 互補濾波 / Kalman 把姿態拉向真實水平
- **就算你放歪了起飛，1~2 秒內也會修正到真水平**

## 你目前程式的問題 ⚠️

你的 [Drone_FC_PID.ino](Drone_FC_PID/Drone_FC_PID.ino) 是**每次開機都重新校水平基準**：

```cpp
levelOffsetRoll  = sumRoll  / N;   // 開機當下的角度當「水平」
```

**問題**：如果你開機時飛機放歪了 5°，程式會把「歪 5°」當成水平 → 飛起來會往一邊偏。

## 正確做法（市售飛控的方式）

| 項目 | 改法 |
|------|------|
| 陀螺儀偏移 | **保持每次開機校**（要靜止）|
| 水平基準 | **改成「只校一次、存 flash」**，之後開機讀回來 |

實作：
1. 用 aux2 觸發校正時，**把 levelOffset 存進 ESP32 flash（Preferences）**
2. 平常開機**只校陀螺儀偏移**，水平基準從 flash 讀回
3. 想重校水平 → 放水平面 → 撥 aux2 → 存新值

這樣：
- 每次開機飛機放歪也沒關係（水平基準是上次在水平面校好存著的）
- 加速度計持續自動修正
- **真正起飛時自穩到「真水平」而不是「你放的角度」**

## 補充：為什麼還需要「水平基準」

理論上加速度計就能自己找水平，為什麼還要校 mounting offset？

因為 **MPU6050 焊在板上、板子黏在機架上，可能本身就裝歪 2~3°**。這個「裝歪」是固定的，校一次存起來即可。校掉之後，飛控看到的「0°」才等於「機架真正水平」。

## 左搖桿

| 推的方向 | 飛機反應 |
|---------|---------|
| **往上推** | 油門變大 → **飛機升高** |
| **往下拉** | 油門變小 → **飛機下降** |
| **往左推** | 機頭**向左轉**（原地轉方向）|
| **往右推** | 機頭**向右轉** |

→ 左手管「**高度**」和「**轉頭方向**」

## 右搖桿

| 推的方向 | 飛機反應 |
|---------|---------|
| **往上推** | 機頭朝下 → **往前飛** |
| **往下拉** | 機頭朝上 → **往後退** |
| **往左推** | 機身左傾 → **往左平移** |
| **往右推** | 機身右傾 → **往右平移** |

→ 右手管「**前後左右移動**」

## 圖示

```
   左搖桿（高度+轉向）        右搖桿（移動方向）

        │                         │
        ↓ 下降                    ↓ 後退
```
