# 四軸無人機 接線製作指南

> 主控板：LOLIN D32（ESP32-WROOM-32）
> 通訊：NRF24L01+PA/LNA
> 動力：3S LiPo + 無刷馬達 × 4 + ESC × 4

---

## 0. 安全規則（任何階段都要遵守）

1. **接線時拔掉電池**，只用 USB 供電做訊號測試
2. **螺旋槳不裝**，直到 PID 自穩程式完成並通過拘束測試
3. **共地必須**：所有 GND 連通，缺一接點就會出現怪現象
4. **電解電容看極性**：接反會爆
5. **NRF24 必須 3.3V**：接 5V 會立即燒掉
6. **燒錄時 ESC 不通電**：USB 供電與電池供電同時開機沒問題，但燒錄中突然斷電會 brick 板子

## 3. 階段 1：MPU6050 姿態驗證（4 條線）

### 接線

```
MPU6050      LOLIN D32
─────────    ──────────
VCC ────────→ 3V（右側最上）
GND ────────→ GND（左側最上）
SDA ────────→ 21（左側）
SCL ────────→ 22（左側）
```

MPU6050 上的 XCL、XDA、AD0、INT 四隻**不接**。

### 軟體設定

[Drone_FC_Gyro.ino](Drone_FC_Gyro/Drone_FC_Gyro.ino) 內：
```cpp
#define ENABLE_NRF24  0
#define ENABLE_ESC    0
```

### 驗證

序列埠 (115200) 應顯示：
```
=== Drone FC boot ===
[1] I2C ready
[2] MPU6050: OK
[3] ESC skipped
[4] NRF24 skipped
=== Ready (disarmed) ===

raw ax= 850 ay= 250 az= 17300 | gx= -540 gy= 210 gz= 35
R   0.7 P  -2.7 Yr   0.3 | ...
```

- 平躺時 az ≈ 16000~17500，ax/ay 接近 0
- 傾斜飛機 R / P 數值跟著動

## 4. 階段 2：加 NRF24 遙控（8 條線）

### 接線

```
NRF24L01+PA/LNA    LOLIN D32
───────────────    ──────────
VCC  ─────────────→ 3V  ⚠️ 不可 5V
GND  ─────────────→ GND
CE   ─────────────→ 4
CSN  ─────────────→ 5
SCK  ─────────────→ 18
MOSI ─────────────→ 23
MISO ─────────────→ 19
IRQ  ─────────────→ 不接
```

### 去耦電容（必裝）

NRF24 模組 VCC ↔ GND 兩腳之間焊一顆 **100µF 電解電容**（C3）：
- 電容 + 腳 → VCC
- 電容 − 腳 → GND

沒這顆 → 發射瞬間電壓崩潰，模組會當機。

### 軟體設定

```cpp
#define ENABLE_NRF24  1
#define ENABLE_ESC    0
```

### 地面端

Nano 燒 [Ground_TX/Ground_TX.ino](Ground_TX/Ground_TX.ino)，沿用你原本 6 通道發射電路。

### 驗證

序列埠：
- `[4] NRF24 listening` 出現
- 動搖桿時 Thr / aux1 / aux2 跟著變
- 把 aux1 開關打到上位 + 油門最低 → `armed=1`、蜂鳴器響（如果你有接）
- aux1 切回 → `armed=0`

## 5. 階段 3：加 ESC × 4（12 條線）

### ESC 接頭是 3-pin（白/紅/黑）

| ESC | 白線 SIG | 紅線 +5V BEC | 黑線 GND |
|-----|---------|------------|---------|
| ESC1 前左 CW | LOLIN **GPIO 25 (DAC1)** | LOLIN **USB**（5V 進） | LOLIN **GND** |
| ESC2 前右 CCW | LOLIN **GPIO 26 (DAC2)** | **包熱縮套不接** | LOLIN **GND** |
| ESC3 後左 CCW | LOLIN **GPIO 27** | 包熱縮套不接 | LOLIN **GND** |
| ESC4 後右 CW | LOLIN **GPIO 14** | 包熱縮套不接 | LOLIN **GND** |

### 動力線（粗紅 + 粗黑，從 ESC 另一端）

```
3S 電池 ──→ XT60 ──→ PDB（電源分配板）
                       │
                       ├─→ ESC1 動力（11.1V）
                       ├─→ ESC2 動力
                       ├─→ ESC3 動力
                       └─→ ESC4 動力
```

⚠️ **粗紅黑線絕對不接 LOLIN**，是 11.1V × 30A，會立刻燒板。

### 軟體設定

```cpp
#define ENABLE_NRF24  1
#define ENABLE_ESC    1
```

### 驗證（不裝螺旋槳）

1. 電池接上 PDB → 4 顆 ESC 嗶嗶聲（識別最低油門）
2. 搖桿 aux1 切上 + 油門 0 → `armed=1`
3. **手撥**每顆馬達轉軸應該轉得動（disarmed 狀態馬達卡住）
4. 慢慢推油門 → 4 顆馬達同步轉
5. 用手感覺**馬達轉向**：M1/M3 順時針、M2/M4 逆時針（從上往下看）

### 馬達轉向錯誤怎麼辦

把 ESC 跟馬達之間的**任 2 條線對調**，方向就反過來。

## 6. 階段 4：加 BMP280 + TCA9548A + VL53L0X（I²C 擴充）

### BMP280（4 條，掛在現有 I²C 主幹）

```
BMP280       LOLIN D32 / 主幹
─────────    ──────────────
VCC ────────→ 3V
GND ────────→ GND
SDA ────────→ 21（與 MPU6050 共用）
SCL ────────→ 22（與 MPU6050 共用）
```

### TCA9548A I²C 多工器（4 + N 條）

```
TCA9548A     LOLIN D32 / 主幹
─────────    ──────────────
VCC ────────→ 3V
GND ────────→ GND
SDA ────────→ 21（與其他 I²C 共用）
SCL ────────→ 22（共用）
A0 / A1 / A2 → 全部接 GND（地址 = 0x70）
RST ────────→ 不接（或接 3V）
```

### VL53L0X × 2 → 接到 TCA9548A 通道

```
VL53L0X #1（前）          TCA9548A
─────────                ─────────
VCC ──────────────────→ 任一 VIN（板上有 3V/5V，看模組規格）
GND ──────────────────→ GND
SDA ──────────────────→ SD0
SCL ──────────────────→ SC0

VL53L0X #2（後）
SDA ──────────────────→ SD1
SCL ──────────────────→ SC1
```

兩顆 VL53L0X **電源都從 TCA9548A 取**或從主板 3V 並聯。

## 7. 階段 5：加 GPS（4 條線）

```
GY-NEO6MV2    LOLIN D32
──────────    ──────────
VCC ─────────→ USB（5V 較穩；3V 也可，看模組標稱）
GND ─────────→ GND
TX  ─────────→ GPIO 16 (RX2)  ← GPS 講話、ESP32 聽
RX  ─────────→ GPIO 17 (TX2)  ← ESP32 設定 GPS 用
```

GPS 天線**朝上**，遠離馬達 / 電源線（避免電磁干擾）。

## 8. 階段 6：加 ESP32-CAM（獨立模組）

ESP32-CAM **不接 LOLIN 任何訊號腳**，只共用電源：

```
ESP32-CAM       電源來源
──────────      ──────────
5V  ──────────→ ESC2 BEC 紅線（取消「包熱縮套」狀態）
                  或 PDB 5V BEC 另一路
GND ──────────→ 共地
```

ESP32-CAM 自己有 WiFi，影像直接傳到電腦。**不需要跟飛控通訊**。

## 9. 電池監測（3 元件分壓）

```
電池正極（11.1V）
        │
        R1 30kΩ
        │
        ├─────→ LOLIN GPIO 34 (A34)
        │
        R2 10kΩ
        │
       GND
```

分壓比 = 10/(30+10) = 1/4
- 滿電 12.6V → ADC 讀 3.15V（在 3.3V 範圍內 ✓）
- 空電 9V → ADC 讀 2.25V

## 11. 製作順序建議

```
週 1：機械
  □ 機架組裝、馬達固定
  □ ESC 焊到馬達（先不裝螺旋槳）

週 2：電源
  □ PDB 焊 XT60
  □ 製作擴充板（電源分配 + 主軌濾波）
  □ 電池監測分壓電路

週 3：飛控
  □ 焊 LOLIN D32 排針
  □ 接 MPU6050（階段 1）→ 燒程式驗證

週 4：通訊
  □ 接 NRF24 + 100µF 電容（階段 2）
  □ 地面 Nano 配對 → 序列埠看到搖桿數值

週 5：動力
  □ 接 4 顆 ESC（階段 3）→ 油門測試（不裝槳！）
  □ 馬達轉向校正

週 6：感測器
  □ 加 BMP280、TCA9548A、VL53L0X（階段 4）
  □ 加 GPS（階段 5）

週 7：影像
  □ 加 ESP32-CAM（階段 6）
  □ 電腦端 WiFi 接收測試

週 8+：PID + 自穩測試
  □ 拘束測試（綁桿子）
  □ 戶外懸停（裝螺旋槳）
```

## 12. 故障排除

| 症狀 | 可能原因 | 解法 |
|------|---------|------|
| 序列埠完全沒輸出 | 監控視窗開太晚 | 按板上 EN/RESET 按鈕 |
| MPU6050: FAIL | I²C 接線錯 / 沒電 | 量 VCC 是否 3.3V，SDA/SCL 沒對調 |
| R/P 數值不變 | 感測器讀不到值 | 看 raw 值是否 0；不為 0 但角度錯 → MPU6050 角度問題 |
| Thr / aux 亂跳 | NRF24 沒接但 ENABLE_NRF24=1 | 改回 0 |
| NRF24 收不到 | 沒裝 100µF 電容 / TX RX 頻道不同 | 加電容，確認 channel 100 一致 |
| ESC 不嗶聲 | 沒收到 1000µs 訊號 | 量 PWM 線、確認 ENABLE_ESC=1 |
| ESC 一通電就尖叫 | 收到非法訊號 | 燒程式時油門可能沒歸 0，重燒並斷電 5 秒再上 |
| 馬達方向反 | 三相線順序錯 | 任 2 條對調 |
| ESP32 重複重開機 | 電源不足 / NRF24 拉爆電 | 加 NRF24 去耦電容 |

## 13. 軟體流程

### 13.1 開發環境準備

**Arduino IDE 設定：**
1. 安裝 Arduino IDE 2.x
2. 偏好設定 → 額外的開發板管理員網址，加入：
   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

3. 開發板管理員 → 搜尋 `esp32` → 安裝 **esp32 by Espressif Systems**
4. 工具 → 開發板 → 選 **WEMOS LOLIN32** 或 **ESP32 Dev Module**

**燒錄速率：** 921600（快）或 460800（穩）

### 13.3 程式檔案結構

```

├── Drone_FC_Gyro/Drone_FC_Gyro.ino     ← 飛機端主程式
├── Ground_TX/Ground_TX.ino             ← 地面端發射器

### 13.4 各階段程式參數

`Drone_FC_Gyro.ino` 頂端有兩個開關控制要啟用什麼：

```cpp
#define ENABLE_NRF24  0    // 0 = 跳過 NRF24，1 = 啟用
#define ENABLE_ESC    0    // 0 = 跳過 ESC，1 = 啟用
```

對應接線階段：

| 階段 | 接什麼 | ENABLE_NRF24 | ENABLE_ESC |
|------|------|--------------|-----------|
| 1. MPU6050 驗證 | 只 MPU6050 | **0** | **0** |
| 2. 加 NRF24 | MPU6050 + NRF24 | **1** | **0** |
| 3. 加 ESC | MPU6050 + NRF24 + ESC | **1** | **1** |
| 4+. 完整版 | 全部 | 改用 `Drone_FlightController.ino` |

⚠️ **接線階段沒做到就把對應旗標設 0**，否則程式會卡在 `radio.begin()` 或亂發 PWM。

### 13.5 燒錄流程

```
1. 用 Micro-USB 線把 LOLIN D32 接到電腦
2. Arduino IDE 工具 → 序列埠 → 選 COMx（通常燒前後 COM 會變）
3. ✓（編譯確認）→ →（上傳）
4. 等待「Hard resetting via RTS pin」出現 = 燒錄成功
5. 工具 → 序列埠監控視窗（鮑率 115200）
6. 按板上 EN/RESET 鈕看開機訊息
```

**燒錄常見問題：**
- 失敗 → 按住 BOOT 鈕再按 RESET（手動進燒錄模式）
- LOLIN D32 通常**不用**按 BOOT，但部分批次需要
- COM 埠抓不到 → 裝 CP2102 / CH340 驅動

### 13.6 序列埠監控操作

開機後正常輸出（階段 1）：
```
=== Drone FC boot ===
[1] I2C ready
[2] MPU6050: OK
[3] ESC skipped (ENABLE_ESC=0)
[4] NRF24 skipped (ENABLE_NRF24=0)
=== Ready (disarmed) ===

raw ax= 850 ay= 250 az= 17300 | gx= -540 gy= 210 gz= 35
R   0.7 P  -2.7 Yr   0.3 | Thr  0 aux10 aux20 | armed0
```

**輸出每行兩個格式輪流：**
- `raw` 行：MPU6050 原始 16-bit 數值（除錯用）
- 第二行：互補濾波後角度 + 遙控器數值 + 武裝狀態

**監控小技巧：**
- 看不到輸出 → 按 EN 重置
- 開機訊息漏掉 → 重置一次
- 中文亂碼 → 看右下角編碼是否 UTF-8

### 13.7 程式邏輯概要

**主迴圈每次跑：**
```
recvData()       讀 NRF24 收到的搖桿資料
failsafe()       檢查 1 秒沒收到就解除武裝
checkArm()       看 aux1 + 油門決定是否武裝馬達
readAttitude()   讀 MPU6050、跑互補濾波算角度
writeMotors()    依油門推 PWM 給 ESC（無 PID）
debugPrint()     每 200ms 印序列埠
```

**武裝邏輯：**
- `aux1=1` + 油門 < 20 → `armed=true`
- `aux1=0` 任何時候 → `armed=false`
- 失聯 > 1 秒 → 強制 `armed=false`

### 13.8 PID 還沒加進來

目前 `writeMotors()` 是「**四顆馬達同油門**」，**不會自穩、不會飛**，只能驗證接線。

PID 自穩程式 → 等硬體全部測試 OK 再加。屆時會在 `writeMotors()` 之前插入：
```
混控公式（X 型四軸）：
  M1 = T + pitchOut + rollOut - yawOut
  M2 = T + pitchOut - rollOut + yawOut
  M3 = T - pitchOut + rollOut + yawOut
  M4 = T - pitchOut - rollOut - yawOut
```

### 13.9 地面端 Nano 程式

`Ground_TX.ino` 跟飛機端 NRF24 設定要**完全一致**：

```cpp
const uint64_t PIPE_OUT    = 0xABCDABCD71LL;   // 必須跟飛機端 PIPE_IN 相同
const uint8_t  NRF_CHANNEL = 100;              // 兩端 channel 必須相同
radio.setDataRate(RF24_250KBPS);                // 兩端速率必須相同
radio.setAutoAck(false);                        // 兩端設定必須相同
```

任一不對應 → 收不到。

### 13.11 程式除錯流程

當行為怪異時依序檢查：

```
1. 接線確認
   □ 用三用電錶量 VCC、GND
   □ I²C 線沒對調

2. 序列埠
   □ 看 [2] MPU6050 是 OK 還 FAIL
   □ 看 raw 值是否合理（az ≈ 16000-17500 靜止）

3. ENABLE 旗標
   □ 沒接的硬體要設 0

4. 編譯訊息
   □ 函式庫版本對嗎
   □ 開發板選對嗎

5. 燒錄
   □ 燒完按 EN 看開機訊息
   □ COM 埠選對

6. 通訊
   □ NRF24 兩端 channel / pipe / 速率必須一致
   □ NRF24 有沒有裝 100µF 去耦電容
```
