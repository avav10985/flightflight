# 飛機端接線總表(V2)

> 主控:LOLIN D32 ESP32-WROOM-32(38 pin)
> 對應程式:`Drone_FC_Full/Drone_FC_Full.ino`
> 跟手把同步契約:NRF24 `Signal` 結構(18 bytes ACK payload telemetry)
> 同類速查文件:`手把接線總表_V2.md`(地面端)

舊版分階段式教學見 `接線指南.md`,本文是「**對照表格速查版**」。

---

## ⚠️ 安全規則(0 階段就要遵守)

1. **接線時拔電池**,用 USB 供電做訊號測試
2. **螺旋槳不裝**,直到 PID 自穩過拘束測試
3. **所有 GND 必共地**,缺一接點都會異常
4. **電解電容看極性**,接反會爆
5. **NRF24 必 3.3V**,接 5V 立燒
6. **燒錄中 ESC 不通電**,USB + 電池可同時,但燒錄中斷電會 brick

## 二、電源樹

```
LiPo 電池 ─┬→ 4 ESC 電池線(粗紅黑並聯)
           ├→ buck IN+/IN−
           │        ↓
           │   buck OUT+ 5V ─┬→ ESP32 5V/VIN 或 USB 腳
           │                 ├→ GPS VCC(看模組 規格,多為 5V)
           │                 └→ 1~2 顆 ESC BEC 5V 接回主軌(其他剪掉)
           │
           └→ 分壓網 ─[30kΩ]─┬─[10kΩ]─ GND
                              │
                              └ GPIO 34(讀電池電壓 × 1/4)

ESP32 3V3(內部 LDO ~500mA)─┬→ NRF24 VCC ⚠️ 3V3 不是 5V
                              ├→ MPU6050 VCC
                              ├→ BMP280 VCC(以後)
                              ├→ GY-271 VCC(以後)
                              ├→ TCA9548A VCC(以後)
                              └→ VL53L0X VCC ×4(以後,經 TCA9548A)
```

⚠️ 4 顆 ESC 的 BEC(中間 5V 紅線)**只保留 1~2 顆接回 ESP32 5V**,其餘剪掉並熱縮包好。多顆 BEC 並聯會互相打架,可能燒。

## 三、各元件詳細接線

### 3.1 NRF24L01+PA/LNA(7 條訊號 + 1 顆濾波電容)

```
NRF24                ESP32
─────                ─────
VCC ─────[100µF + 0.1µF 並聯]─── 3V3      ⚠️ 不能 5V
GND ─────────────────────────── GND
CE  ─────────────────────────── GPIO 4
CSN ─────────────────────────── GPIO 5
SCK ─────────────────────────── GPIO 18
MOSI ────────────────────────── GPIO 23
MISO ────────────────────────── GPIO 19
IRQ ─── 不接(浮空)
```

**100µF + 0.1µF 並聯**焊在 NRF24 模組底部 VCC↔GND,降低 PA/LNA 大電流尖峰。

### 3.2 ESC × 4(每顆 3 條訊號 + 動力線)

| ESC | 馬達位置 | 轉向 | 訊號接 GPIO |
|--|--|--|--|
| ESC1 | M1 前左 | CW 順 | **25** |
| ESC2 | M2 前右 | CCW 逆 | **26** |
| ESC3 | M3 後左 | CCW 逆 | **27** |
| ESC4 | M4 後右 | CW 順 | **14** |

每顆 ESC 3-pin BEC 線:
```
白 / 黃(訊號) ── ESP32 GPIO 25/26/27/14
紅           ── 5V(1~2 顆連接、其他剪)
黑           ── GND(全部連)
```

動力線(粗紅黑,從 ESC 另一端輸出到電池):
```
粗紅 ── 電池 +(全部 4 顆並聯)
粗黑 ── 電池 −(全部 4 顆並聯)
```

### 3.3 MPU6050(4 條,你已接好 ✓)

```
MPU6050     ESP32
────────    ─────
VCC ─────── 3V3 或 5V(GY-521 模組板上有 LDO,兩者都行)
GND ─────── GND
SDA ─────── GPIO 21
SCL ─────── GPIO 22
INT ─── 不接
AD0 ─── 不接(浮空 / 接 GND → I²C 位址 0x68 預設)
```

> ⚠️ 只有「GY-521 模組」可以接 5V,裸 MPU6050 IC 只能 3V3(超壓會燒)。
> GY-521 板上 SDA/SCL 上拉電阻接內部 3V3(LDO 後),即使 VCC 是 5V,
> I²C 線電位還是 3V3,跟其他 I²C 元件共線安全。

### 3.4 GPS NEO-6M(4 條 + 天線)

```
GPS         ESP32
───         ─────
VCC ─────── 5V(部分模組 3V3,看模組標籤)
GND ─────── GND
TX ──────── GPIO 33(ESP32 接收)   ⚠️ 2026-06-07 從 UART2 預設(16/17)搬到 32/33
RX ──────── GPIO 32(ESP32 傳送)
```

> ESP32 UART 透過 GPIO matrix 可以指定任何 GPIO 當 RX/TX,不受預設限制。
> 程式 `Serial2.begin(9600, SERIAL_8N1, PIN_GPS_RX, PIN_GPS_TX)` 兩個參數對應 GPIO 33 跟 32。
> 副作用:GPIO 16/17 釋出回 「可用腳」清單。

NEO-6M 冷啟動 30~120 秒(戶外天空下測)。

### 3.5 電池電壓監測(3 元件分壓)

```
電池 +B ──┬─[30kΩ]──┬──[10kΩ]── GND
          │          │
          │          └── GPIO 34(ADC 讀分壓中點)
          │
          (粗線接 ESC + buck IN)
```

11.1V 電池滿電 → 中點 2.78V → ADC ~3450 → 程式 `BAT_DIVIDER = 4.0` 還原成 11.1V。

## 四、之後加(`ENABLE_XXX` 旗標 0,程式骨架已有)

### 4.1 GY-271 磁力計(2026-06-06 新加,要先買!)

```
GY-271     ESP32
────────   ─────
VCC ────── 3V3
GND ────── GND
SCL ────── GPIO 22(共 I²C)
SDA ────── GPIO 21(共 I²C)
DRDY ─── 不接(浮空)
```

啟用:`#define ENABLE_HMC5883 1` + Serial 送 `magcal` 校準 10 秒繞圈。

### 4.2 BMP280 氣壓計

```
BMP280     ESP32
─────      ─────
VCC ────── 3V3
GND ────── GND
SCL ────── GPIO 22(共 I²C)
SDA ────── GPIO 21(共 I²C)
CSB ─── 不接(I²C 模式)
SDO ─── GND(位址 0x76)或 VCC(0x77)
```

啟用:`#define ENABLE_BMP280 1`。

### 4.3 VL53L0X × 4 + TCA9548A I²C 多工器

```
TCA9548A     ESP32
────────     ─────
VCC ──────── 3V3
GND ──────── GND
SCL ──────── GPIO 22(共 I²C)
SDA ──────── GPIO 21(共 I²C)
A0/A1/A2 ─── GND(位址 0x70)
RESET ────── 不接(內部上拉)

  ↓
TCA9548A CH0 SDA/SCL → VL53L0X #1(前中)SDA/SCL
TCA9548A CH1 SDA/SCL → VL53L0X #2(前左)SDA/SCL
TCA9548A CH2 SDA/SCL → VL53L0X #3(前右)SDA/SCL
TCA9548A CH3 SDA/SCL → VL53L0X #4(底部)SDA/SCL

VL53L0X 都 VCC → 3V3、GND → GND(可共軌)
```

啟用:`#define ENABLE_VL53L0X 1`。閾值:前 3 顆任一 < 1.8m 觸發避障,底部 < 30cm 觸發降落。

## 4.4 I²C 主幹長線材(2026-06-07 加)

如果 ESP32 → TCA9548A 主幹超過 20cm(機架大,中央 ESP32 位置遠
離 TCA9548A 安裝點),要做以下三件事:

1. **雙絞線**(SDA + SCL 互相纏繞,每 1cm 一圈)
2. **I²C 速度降到 100 kHz**:
   ```cpp
   Wire.setClock(100000);   // 2026-06-07 已改
   ```
3. **遠離 ESC / 馬達電源線 ≥ 5cm**

I²C 主幹元件位址全表:

| 位址 | 元件 |
|--|--|
| 0x68 | MPU6050 |
| 0x76 | BMP280(SDO 接 GND)|
| 0x1E / 0x0D | HMC5883L / QMC5883 磁力計 |
| 0x70 | TCA9548A(A0/A1/A2 全 GND)|

⚠️ 不要讓 TCA9548A A0/A1/A2 設成 0x76 或 0x77,會跟 BMP280 衝突。

## 五、I²C 匯流排 共用情況

GPIO 21/22 上會同時接多個 I²C 元件,各自有不同位址:

| 元件 | I²C 位址 | 何時加入 |
|--|--|--|
| **MPU6050** | 0x68(AD0=GND)或 0x69(AD0=VCC)| ✅ 第一階段 |
| BMP280 | 0x76 或 0x77 | `ENABLE_BMP280=1` |
| HMC5883L 磁力計 | 0x1E | `ENABLE_HMC5883=1`(自動偵測)|
| QMC5883 中國版 | 0x0D | 同上 |
| TCA9548A 多工器 | 0x70 | `ENABLE_VL53L0X=1` |

I²C 主幹要 **4.7kΩ 或 10kΩ 上拉電阻 × 2**(SDA、SCL 各一),拉到 3V3。**很多模組自己已經有上拉電阻**,如果有 2~3 個模組接在一起且每個都有上拉,**等於並聯阻抗更低**,有時候會干擾通訊。

如果 I²C 通訊不穩,**剪掉部分模組的上拉電阻**(模組背面通常有焊點 / 跳線),只留主幹一組 4.7kΩ 上拉。

## 七、接的順序建議

1. **GND 軌**(全系統共地)
2. **電池 + buck → 5V 軌**(量到穩 5V 才繼續)
3. **3V3 軌**(ESP32 LDO 出來)
4. **MPU6050** ← 你已完成
5. **NRF24**(加 100µF 電容)→ 立刻能跟手把對通
6. **GPS** → Serial 看到衛星數
7. **4 顆 ESC**(不裝槳,純電測命令)
8. **電池分壓 ADC34**
9. (以後)GY-271 → BMP280 → VL53L0X×4 + TCA9548A

## 七之二、ESP32-CAM(影像,獨立電源 + WiFi)

ESP32-CAM **不接到 主 ESP32 任何訊號腳**,只需要電源:

```
ESP32-CAM        接到
─────────        ────
5V (VCC) ──→ ESC BEC 5V(獨立,**不從 buck 5V 軌拉**)
GND      ──→ 共 GND 軌(跟主系統一起)
天線     ──── 朝下避開機架(或外接 SMA 天線)
```

### 為什麼從 ESC BEC 拿電,不從 buck?

1. **隔離負載**:CAM WiFi 峰值 ~500mA,單獨用 ESC BEC 不會拖累主 5V → 不會 BOD
2. **保護主控**:CAM 短路只燒 ESC BEC,主 ESP32 不受影響
3. 只需要主系統 GND 共軌就行

### 影像怎麼看

**手機瀏覽器**(暫定):
1. 飛行前手機開 WiFi 熱點(SSID/密碼存在 `ESP32CAM_Flight/secrets.h`,已 gitignore)
2. ESP32-CAM 上電自動連手機熱點
3. Serial 印出 IP,手機瀏覽器開那個 IP → 即時影像
4. 或 ESP32-CAM 自己開 AP,手機連它

### 未來可選:手把 TFT 看

技術可行(ESP32-S3 內建 WiFi + 8MB PSRAM),要寫 ~100~150 行程式
(WiFi 抓 JPEG + 解碼 + TFT 推圖)。**先飛起來再說**,影像之後加。

### ⚠️ WiFi 跟 NRF24 干擾

兩個都在 2.4GHz,**會互擾**:
- NRF24 控制距離可能縮短 20~30%
- CAM frame rate 也會降
- 短距離飛行(< 30m)感受不明顯
- 真要長飛先關 CAM,只用 NRF24 控制

## 八、驗證

每階段接完都這樣驗:
- **電壓量**:模組 VCC、GND 對 ESP32 GND 量到該有的電壓
- **Serial 印**:`Drone_FC_Full` 開機訊息有看到對應的「[+] XXX OK」
- **沒短路**:斷電量 VCC ↔ GND 阻抗 > 1kΩ(短路會 < 100Ω)

關於各階段詳細驗證流程,看 `接線指南.md` 階段 1~6 章節。
