A.1系統拓樸

系統由三個 MCU 節點構成:地面站(ESP32-S3)、飛行控制器(ESP32)與獨立影像節點(ESP32-CAM)。主控制鏈路使用 nRF24L01+ 2.4 GHz 專屬通道;語音辨識與影像串流使用 WiFi,與控制鏈路在頻段上分時使用以避免干擾。點擊下圖任一元件可檢視其技術說明與設計考量。

A.2程式架構

A.2.1 程式組織策略

專案採「主程式 + 單元測試程式」雙層結構:兩支主程式(Drone_FC_Full 飛行控制器、Ground_TX_ESP32 地面站)承載完整功能;另維護 28 支獨立可燒錄的單元測試程式,每個子系統(IMU、GPS、TFT、I²S 音訊、WiFi、NRF24⋯)均有對應之最小化驗證程式。系統級問題發生時,可立即降階至單元程式重現,大幅縮小問題定位範圍 —— 本專案多數硬體問題(如穩壓器容量不足)均以此方法確診。

A.2.3 地面站程式結構

  • 50 Hz 控制迴圈:搖桿 ADC 取樣(含反相校正與中位映射)→ 開關狀態編碼 → 組裝 Signal 封包 → nRF24 發送 → 讀取 ACK payload 遙測。
  • TFT 介面狀態機:兩個畫面(飛行資訊畫面 / PID 調參選單)。繪製採 dirty-flag 策略:靜態框架只畫一次,動態數值僅於變動時重繪 —— 避免每迴圈全螢幕重繪佔用共用 SPI 匯流排、拖慢 nRF24 通訊。畫面切換時設置 g_flightDirty / g_menuDirty 旗標強制完整重繪一次。
  • 選單操作:選單鍵進入 / 離開,搖桿上下移動游標、左右調值,共 8 個項目(5 個 PID 增益 + 3 個動作項,詳見飛控與校準章)。
  • 編譯期功能旗標:ENABLE_SD / ENABLE_VOICE_MODE10 / ENABLE_MUSIC / ENABLE_PC_MODE11 等,未完成或暫停用之功能以旗標隔離,不影響主功能編譯與穩定性。飛機端同樣以 ENABLE_BMP280 / ENABLE_VL53L0X / ENABLE_HMC5883 控制感測器啟用。
  • 憑證隔離:WiFi 帳密與 API 金鑰集中於 secrets.h(排除於版本控制),倉庫提供 secrets.h.example 範本。
  • 持久層:飛機端以 ESP32 NVS 儲存 IMU 校準與 PID 參數,斷電保留、開機自動載入。

A.3通訊協定

控制鏈路採 nRF24L01+ Enhanced ShockBurst,通道 88(2.488 GHz),自訂 5-byte 管線位址(避開函式庫常見預設值,降低同頻干擾)。地面站以約 50 Hz 發送指令封包;遙測資料以 ACK payload 機制搭載於確認封包回傳,不佔用額外傳輸時隙。

A.3.1 下行封包 Signal(地面站 → 飛行控制器,12 bytes)

欄位型別說明
throttle / pitch / roll / yawuint8 × 4搖桿值 0–255,中位 127
modeuint8飛行模式兩位數編碼(詳見飛控與校準章)
flagsuint8bit 0:完整校準觸發;其餘保留
paramIDuint8線上調參 / 遠端動作識別碼(0 = 無)
paramValfloat參數值(paramID ≠ 0 時有效)

A.3.2 上行封包 Telemetry(飛行控制器 → 地面站,20 bytes)

欄位型別說明
roll / pitchint16 × 2姿態角 × 10(0.1° 解析度)
yawRateint16偏航角速度 × 10(°/s)
altitude_dmint16相對高度(分米),範圍 ±3276.7 m
headingint16絕對航向 × 10(0–3599,0 = 北)
statusuint8位元旗標:armed / 校準中 / 校準失敗 / GPS fix
satCountuint8GPS 可見衛星數
lat_e7 / lon_e7int32 × 2經緯度 × 10⁷(約 1.1 cm 解析度)

封包大小 20 bytes,符合 nRF24 ACK payload 上限(32 bytes)並保留擴充空間。失聯保護:訊號中斷超過 1 秒,飛行控制器立即解除武裝並將油門歸零(詳見飛控與校準章)。
