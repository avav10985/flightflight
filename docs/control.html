CHAPTER C

飛控與校準

飛行模式、安全機制、姿態控制與 IMU 校準系統

C.1飛行模式與安全機制

C.1.1 模式編碼

模式由地面站兩顆三段開關組合產生:mode = SW_A × 10 + SW_B(各 0 / 1 / 2,撥桿上 = 0、中 = 1、下 = 2),共 9 個有效值。採兩位數編碼而非連續整數 0–8,使十位 / 個位各自保留語意(十位 = 控制來源類別,個位 = 子模式),封包僅佔 1 byte。

編碼名稱說明狀態
00安全 / 校準馬達強制鎖定;進入瞬間自動觸發快速校準;可執行完整校準與 PID 調參(§C.4)已完成
01手動自穩搖桿設定目標角度,PID 維持姿態 —— 目前唯一允許武裝之模式已完成
02GPS 定點導航自動飛向目標座標;距離 / 方位角計算與導航狀態機完成,待接入 PID 設定點部分完成
10語音控制雲端 STT + LLM 指令解析(語音子系統章);辨識管線已驗證,待接入控制鏈部分完成
11PC 控制USB Serial 透明橋接,由電腦端程式控制規劃中
12 20 21 22預留——

C.1.2 武裝(Arm)安全鏈

馬達輸出受四層獨立條件保護,任一不滿足即解除武裝(disarm)並鎖定馬達:

層級機制設計目的
1. 開機鎖定開機後 safetyReleased = false,必須先撥至 mode 00 一次才解除鎖定防止開機時開關停在飛行模式、油門非零造成馬達直接啟動
2. 模式白名單modeFlyable() 函式明確列舉可飛模式(目前僅 01);未實作模式一律拒絕武裝新模式必須顯式加入白名單,預設安全
3. 油門條件解鎖後仍須油門收底(< 20 / 255)才完成武裝防止切模式瞬間以高油門起轉
4. 失聯保護超過 1 秒未收到地面站封包 → 立即 disarm、油門歸零、PID 積分項清除遙控失效時不令飛機亂飛;disarm 同時將懸停學得之水平偏移寫入 NVS(§C.4.4)

C.2姿態估計 — 互補濾波

陀螺儀積分短期精確但長期漂移;加速度計長期穩定但受機體振動干擾。互補濾波以高通 / 低通互補方式融合兩者,每迴圈執行:

roll₀ = 0.98 × (roll + gyro_roll × dt) + 0.02 × acc_roll pitch₀ = 0.98 × (pitch + gyro_pitch × dt) + 0.02 × acc_pitch

係數 0.98 / 0.02 之物理意義:陀螺積分主導瞬時響應(高通),加速度計以約 1–2 秒時間常數緩慢拉回絕對基準(低通),振動雜訊被平均掉。偏航軸無重力參考,採相同結構但以磁力計絕對航向作為 2% 修正源;無磁力計時退化為純積分模式(可飛但航向緩慢漂移)。

C.2.1 實作程式碼(節錄自 Drone_FC_Full.ino)

// 每迴圈執行:gRoll/gPitch 為陀螺角速度(°/s),accRoll/accPitch 為 // 加速度計反推的絕對角度。dt 為與上一迴圈的實際時間差(秒)。 roll = 0.98f * (roll + gRoll * dt) + 0.02f * accRoll; pitch = 0.98f * (pitch + gPitch * dt) + 0.02f * accPitch;
說明:第一項「上次角度 + 陀螺角速度 × dt」是積分推算的新角度(反應快、會漂移);第二項是加速度計算出的絕對角度(不漂移、但有振動雜訊)。以 98:2 加權後,兩者缺點互相抵銷。

C.3PID 姿態控制

C.3.1 控制律

角度環 PID:搖桿值線性映射為目標角度,誤差經 PID 產生修正量,再進入 X 型混控:

setpoint = (stick − 127) × (MAX_ANGLE / 127) // MAX_ANGLE = 20° output = Kp·e + Ki·∫e·dt + Kd·de/dt // e = setpoint − 量測角 M1(前左 CW) = T + pitchOut + rollOut − yawOut // 混控 M2(前右 CCW) = T + pitchOut − rollOut + yawOut M3(後左 CCW) = T − pitchOut + rollOut + yawOut M4(後右 CW) = T − pitchOut − rollOut − yawOut // 各輸出限幅 [1000, 2000]

C.3.1.1 實作程式碼(節錄自 Drone_FC_Full.ino)

// 搖桿(0~255,中位 127)線性映射為目標角度,MAX_ANGLE = 20° float rollSet = (data.roll - 127) * (MAX_ANGLE / 127.0f); float pitchSet = (data.pitch - 127) * (MAX_ANGLE / 127.0f); // X 型混控:油門為基準,三軸 PID 修正量依馬達位置與旋向疊加 int m1 = baseT + pitchOut + rollOut - yawOut; // 前左 CW int m2 = baseT + pitchOut - rollOut + yawOut; // 前右 CCW int m3 = baseT - pitchOut + rollOut + yawOut; // 後左 CCW int m4 = baseT - pitchOut - rollOut - yawOut; // 後右 CW m1 = constrain(m1, 1000, 2000); // 限幅,其餘三顆同
說明:想前傾(pitch)時前排馬達減速、後排加速;想右滾(roll)時左排加速、右排減速;偏航(yaw)靠對角線馬達轉速差產生的反扭矩 —— 正負號完全由各馬達的位置與螺旋槳旋向決定。

C.3.2 調參歷程

PID 增益經三輪實飛迭代,過程記錄如下 —— 這是全專案耗時最多的調試項目之一:

輪次現象調整
初版輕推搖桿馬達即急遽加速,姿態過衝後振盪 —— 增益過高,室內無法安全測試Kp / Kd 全面下調至保守值,最大傾角限制 15°
第二輪保守值下姿態穩定,但操縱響應遲鈍,室外抗風不足最大傾角 15° → 20°(中等響應);增益維持保守,改以線上調參逐步試探
現行Kp 1.0 / Ki 0.01 / Kd 0.3(roll & pitch 共用),yaw 角速度環 Kp 0.8 / Ki 0.01作為基準值寫入 NVS;後續微調全部透過手把選單線上進行,毋須重新燒錄

C.3.3 線上調參管線

調 PID 最痛的傳統流程是「改程式 → 燒錄 → 試飛 → 再改」。本系統把調參做成遙控線上操作,試飛中即可即時改值:

  1. 手把選單:選單鍵進入,搖桿上下移游標、左右調值。8 個項目:Kp / Ki / Kd / KpY / KiY(增量分別 0.1 / 0.005 / 0.05 / 0.1 / 0.005)+ 三個動作項「保存PID」「快速校準」「完整校準」。
  2. 封包傳遞:變動值以 paramID(1–5 = 增益、100 / 101 / 102 = 動作)+ paramVal 搭載於 50 Hz 控制封包,不中斷飛行控制。
  3. 飛機端去重:50 Hz 下同一參數會重複收到數十次,接收端以「ID + 值未變化即忽略」去重,動作項(校準 / 存檔)不致重複觸發。
  4. 持久化:調到滿意後選「保存PID」(paramID 102),飛機端將五個增益寫入 NVS;重新上電自動載入,毋須每次重調。

C.4IMU 校準系統

校準是本專案實際飛行品質的決定性因素,也是除錯耗時最久的子系統。核心問題:MPU-6050 的陀螺零偏隨溫度漂移、模組焊接與機架組裝必然存在水平安裝誤差 —— 水平基準偏差 1°,飛機起飛就會持續往該方向橫移(問題 I-08「起飛偏一邊」之根因)。校準系統因此演化出三層機制。

C.4.1 兩級校準

快速校準(gyro)完整校準(full)
校準對象陀螺儀零偏(三軸靜止讀值平均)陀螺零偏 + 水平基準(加速度計姿態零點)
機體要求靜止即可,姿態不拘必須置於水平面 —— 校準面即未來的「水平」定義
耗時約 1 秒數秒
觸發時機每次進入 mode 00 自動執行手動觸發(換場地 / 摔機後 / 起飛持續偏移時)

C.4.2 觸發路徑

兩種校準均可遠端觸發,且僅於未武裝 + mode 00 + 油門收底時受理:

路徑機制觸發
模式邊緣自動偵測 mode 非 0 → 0 之轉換瞬間快速校準
手把選單paramID 100 / 101(含去重)快速 / 完整校準
flags 位元Signal.flags bit 0 之 0→1 邊緣完整校準
序列埠指令cal / calfull(開發除錯用)快速 / 完整校準

C.4.3 動作偵測拒寫

早期版本曾因校準期間手持晃動,把錯誤的零偏與水平基準寫入,導致後續飛行全面異常。現行校準程序全程監測量測波動,超出閾值即判定校準失敗、拒絕寫入並回報地面站(Telemetry status 之 CAL_FAILED 位元,TFT 顯示),保留前次有效校準值。

C.4.4 懸停水平自學習(in-flight level learning)

即使完整校準成功,校準平面與真實水平仍可能有殘餘誤差(桌面本身不平)。系統因此加入飛行中緩慢自學習機制:

// 進入條件(需同時成立並持續 3 秒): // armed + 油門位於懸停區 + 偏航角速度 < 5°/s // + 角速度變化 < 5°/s + |加速度向量| ≈ 1 G(±7%,代表未加減速) levelOffsetRoll += roll × 0.0002 // 每迴圈吸收當前殘餘角的萬分之二 levelOffsetPitch += pitch × 0.0002 // 50 Hz 下約 100 秒完全收斂
  • 學習速率刻意設定極慢:太快會與 PID 自穩互相拉扯,太慢使用者無感;0.0002 / 迴圈為實測折衷。
  • 嚴格的進入條件確保只在「真懸停」時學習 —— 機體加減速或轉向期間之姿態角不會被誤認為水平偏差。
  • 降落 disarm 或失聯 disarm 時,學得之偏移自動寫入 NVS:每次飛行都讓下次起飛更準。

C.4.5 持久化(NVS)

陀螺零偏、水平基準、自學習偏移與五個 PID 增益均存於 ESP32 NVS(非揮發性快閃儲存),開機自動載入。實務效益:野外換電池重開機,不需要找水平面重新校準即可起飛。
