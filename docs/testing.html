CHAPTER E

測試與驗證

問題解決紀錄、未來工作與參考資料

E.1問題解決紀錄

下表彙整開發期間之系統級問題、根因分析方法與處置結果。多數問題之定位採對照實驗法:固定其餘變因、單獨替換疑似元件,以重現性確認根因。

編號問題現象根因分析處置狀態
I-01 ESP32-S3 執行 WiFi 初始化即無限重啟,最簡 5 行測試程式亦重現 同一 USB 線 / 同一程式對照測試:經典 ESP32(AMS1117 1 A)正常、S3 模組必然重啟 → 鎖定 S3 板載穩壓器(約 300 mA)不足以供應 WiFi RF 啟動電流(約 400 mA) 更換為 1 A 穩壓器之開發板,並增設 LD1117 獨立周邊軌(硬體設計章) 已結案
I-02 SD 卡模組接入後 nRF24 通訊完全失效 SD 模組之電平轉換晶片於 CS 未選取時仍驅動 MISO 線,佔用共用 SPI 匯流排 暫時停用 SD;電源架構更新後重新驗證中,必要時改用無電平轉換之 3.3 V 裸模組 驗證中
I-03 nRF24 以最大發射功率工作時系統偶發重啟 PA_MAX 發射瞬間電流尖峰致穩壓器輸出電壓下垂 發射功率暫降至 PA_MIN(桌面測試距離足夠);電源改善後將復測高功率檔位 已緩解
I-04 喇叭於開機及待機時輸出雜訊 MAX98357A shutdown 腳浮空導致擴大機隨機致能,放大 I²S 匯流排時脈串擾 開機即將 SD 腳設為輸出並拉低,播放時方致能 已結案
I-05 雲端推論服務回應 HTTP 429(速率限制) 免費層級額度(15–30 請求/分鐘)無法支撐連續整合測試 STT 與 LLM 解析拆至不同服務之獨立額度;中期規劃改用本地離線辨識(§E.2) 已緩解
I-06 語音正確轉錄為「起飛」但 LLM 回傳 unknown STT 輸出附帶標點符號,與 few-shot 範例字面失配 解析前正規化:去除前後空白與中英文標點 已結案
I-07 ESP32-S3 原生 USB CDC 序列埠於部分主機列舉不穩 S3 原生 USB 列舉時序與主機端驅動相依 所有執行期狀態改以 TFT 顯示,序列埠僅作燒錄輔助(與電源互斥限制一致) 已結案
I-08 起飛瞬間機體向固定方向偏移 IMU 水平基準偏差(校準時機體未完全水平);偏差 1° 即造成持續橫移 三層對策:完整校準程序 + 動作偵測拒寫 + 懸停水平自學習(飛控與校準章 §C.4) 持續觀察

E.2未來工作

項目內容優先序
本地離線語音辨識導入 Espressif ESP-SR(AFE + MultiNet),以拼音定義中文命令詞於裝置端直接辨識;消除網路依賴、速率限制與雲端延遲(目標 < 1 s)。需自行打包中文模型(mn6_cn)至 flash model 分區高
語音指令 → 控制鏈路銜接將 Mode 10 解析所得 action 轉換為 Signal 封包序列,經 nRF24 注入飛行控制(含逾時回復搖桿控制之安全機制)高
GPS 導航閉環Mode 02 導航輸出(目標 roll / pitch)接入 PID 設定點,完成定點飛行閉環;需戶外定位精度驗證中
氣壓定高BMP280 高度估計 + 油門軸 PID,實現高度保持;為自動降落之前置功能中
SD 卡資料記錄新電源架構下重新驗證 SPI 共存;啟用後支援飛行紀錄與音訊播放中
頭部追蹤雲台手機 IMU(DeviceOrientation API)→ WiFi → 機載雙伺服雲台;原型已驗證,待整合至主程式低
