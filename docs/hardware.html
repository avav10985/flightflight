B.0.2 組裝方式

  • 飛機:輕量機架,飛控板、感測器與馬達驅動以洞洞板焊接整合;感測器統一掛載於 I²C 匯流排減少佈線。
  • 地面站(三板架構):A 板(ESP32-S3 主控)、B 板(TFT 螢幕,4–7 條副排線連接)、C 板(被動元件樞紐 —— 全部分壓電阻與肩鍵集中,主板佈線單純化)。搖桿、開關、按鈕安裝於面板,訊號線匯入 C 板。
  • 佈線迭代:接線設計以文件(接線總表)先行、實作驗證後回寫修正,共迭代十餘版;GPIO 配置曾為佈線整齊度而對調(左右搖桿訊號各自集中同側)。
  • 設計圖產生方式:電路 / 接線 / 板面配置圖均以 Python 腳本程式化產生 —— 改一條線即重跑腳本,確保圖面與實體同步。

B.1地面站電源架構

初版設計中所有 3.3 V 負載均由 ESP32-S3 模組板載穩壓器供電。實測發現該穩壓器(額定約 300 mA)無法承受 WiFi 射頻啟動之瞬時電流(約 400 mA),導致欠壓重啟(brown-out)。重新設計後採雙穩壓器分軌架構:晶片與周邊各自擁有獨立 3.3 V 軌,僅共地,射頻電流尖峰不再影響周邊供電品質。

圖 B-1 地面站雙穩壓器分軌電源架構

本架構之直接效益:WiFi 啟動與發射期間,周邊軌電壓不受晶片軌負載變動影響,TFT 顯示器閃爍與 nRF24 通訊不穩定問題隨之消除。

操作程序影響:韌體燒錄時僅接 USB(電池斷開),飛行與整合測試時僅接電池(USB 斷開)。此限制意味序列埠監看與周邊(TFT / 麥克風)無法同時使用 —— 故所有執行期狀態與錯誤訊息均設計為顯示於 TFT。

B.2地面站腳位配置(ESP32-S3)

SPI 匯流排由 TFT 與 nRF24L01+ 共用(bus_shared 模式,各自獨立 CS);microSD 因其模組電平轉換晶片會佔用 MISO 線(問題 I-02),移至獨立之第二組 SPI(SPI3)。I²S 匯流排由麥克風與擴大機共用 BCLK / WS 時脈。
