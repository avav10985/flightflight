1.摘要

本專案自零開始設計並實作一套完整的四軸無人機系統,不採用任何現成飛控板或遙控器,涵蓋三個子系統:

  • 飛行控制器:以 ESP32 為核心,自行實作慣性量測融合(互補濾波)、三軸 PID 姿態控制、X 型四馬達混控,並整合 GPS、氣壓計、磁力計與雷射測距感測器。
  • 地面站(手把):以 ESP32-S3 為核心,2.8 吋彩色 TFT 全中文操作介面,雙搖桿 + 9 種飛行模式,經 nRF24L01+ 與飛行控制器進行 50 Hz 雙向通訊(指令下行 / 遙測上行),支援線上 PID 調參與遠端校準。
  • AI 語音控制子系統(Mode 10):地面站內建 I²S 麥克風,按住肩鈕錄音後經 WiFi 上傳雲端,以 Whisper 模型進行中文語音辨識,再由大型語言模型解析為結構化飛行指令(JSON),全程延遲約 3 秒。

開發過程採對照實驗方法定位多項系統級問題,包括穩壓器容量不足導致之 WiFi 欠壓重啟、SPI 匯流排佔用衝突、射頻發射電流尖峰等,並完成雙穩壓器電源架構之重新設計。本報告依主題分章,記錄系統之完整技術規格、設計決策與驗證結果。

2.作品功能

本作品為一套可實際飛行之四軸無人機與其地面控制系統,所有功能均為自行設計實作:

功能說明狀態
姿態自穩飛行自寫飛控:互補濾波姿態估計 + 三軸 PID 閉環控制,搖桿放開自動回水平。PID 控制原理參考 [4]可飛行
無線遙控與遙測nRF24 2.4 GHz 雙向鏈路:50 Hz 指令下行,姿態 / 高度 / 航向 / GPS 即時回傳並顯示於手把彩色螢幕已完成
線上 PID 調參與遠端校準飛行中透過手把選單即時調整控制增益、遠端觸發 IMU 校準,結果斷電保存已完成
AI 語音控制對手把說「起飛」「向前飛三秒」,雲端語音辨識 + 大型語言模型解析為飛行指令。設計參考 ESP32 語音助理相關作品 [1][2][3]辨識管線完成
GPS 定點導航自動飛向指定座標:導航計算與狀態機完成,待接入控制環部分完成
第一人稱即時影像機載 ESP32-CAM WiFi 串流,飛行中可調畫質;頭部追蹤雲台原型已驗證已完成
多層飛行安全機制開機鎖定、模式白名單、油門武裝條件、1 秒失聯自動斷電已完成

3.系統規格總覽

3.1 飛行控制器(機載)

項目規格
主控制器LOLIN D32(ESP32-WROOM-32,雙核心 Xtensa LX6 @ 240 MHz)
慣性量測單元InvenSense MPU-6050(三軸陀螺儀 + 三軸加速度計,I²C @ 100 kHz)
衛星定位u-blox NEO-6M(GY-NEO6MV2,UART @ 9600 bps,NMEA)
氣壓高度計Bosch BMP280(I²C 0x76,相對高度估計)
磁力計QST QMC5883 / QMC5883P(GY-271,自動偵測位址 0x0D / 0x2C)
雷射測距ST VL53L0X × 2,經 TCA9548A I²C 多工器(0x70)
動力空心杯馬達 × 4,MOSFET PWM 直驅,X 型配置
無線通訊Nordic nRF24L01+(SPI,CH 88,自訂 5-byte pipe address)
第一人稱影像ESP32-CAM(獨立 MCU,WiFi MJPEG 串流,固定 IP + mDNS)

3.2 地面站(手把 V2-A)

項目規格
主控制器ESP32-S3 N16R8(雙核心 LX7 @ 240 MHz,16 MB Flash,8 MB Octal PSRAM)
顯示器2.8" TFT MSP2806(ILI9341) 240×320(SPI @ 40 MHz,LovyanGFX,中文字型)
操縱輸入雙軸搖桿 × 2(油門軸移除回中彈簧)、三段開關 × 2、肩鈕 × 2、選單鍵
語音輸入INMP441 I²S MEMS 麥克風(16 kHz / 16-bit / 單聲道)
語音輸出MAX98357A I²S Class-D 擴大機(預留 TTS 語音回覆)
資料儲存microSD(SPI,與 TFT / nRF24 共匯流排)
電源2S 鋰電池 → 降壓模組 5 V → 雙 LDO 分軌(詳見硬體設計章)
