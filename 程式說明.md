# 四軸無人機 飛控程式說明報告

> 專案：ESP32 (LOLIN D32) 四軸無人機
> 主控：LOLIN D32（ESP32-WROOM-32）
> 遙控：Arduino Nano + NRF24L01+PA/LNA

---

## 1. 概述

本專案的飛控程式採「**分階段開發**」設計，每個階段一支獨立程式，
功能由簡到繁逐步疊加。每通過一個階段的驗證，才進入下一支程式，
目的是讓除錯範圍縮小、避免一次堆太多功能找不到問題。

飛機端程式共 4 支（依階段）：

```
Gyro ──→ Manual ──→ PID ──→ Full
階段1     階段2      階段3    階段4
```

另有地面端發射器程式 1 支與多支感測器獨立測試程式。

---

## 2. 飛機端飛控程式

### 2.1 Drone_FC_Gyro（階段 1：感測器驗證）

| 項目 | 內容 |
|------|------|
| **目的** | 驗證 MPU6050 姿態感測器接線正確、讀數合理 |
| **硬體** | LOLIN D32 + MPU6050（I²C）|
| **功能** | 讀加速度計 + 陀螺儀、互補濾波算出 Roll/Pitch、序列埠輸出 |
| **不含** | NRF24、ESC、PID |

**用途說明：**
最基礎的程式，只做一件事——確認 MPU6050 能正確讀到姿態。
開機自動校正陀螺儀偏移，並用「互補濾波」融合加速度計與陀螺儀，
輸出 Roll（翻滾角）、Pitch（俯仰角）、Yaw 角速度。
此階段用來確認感測器軸向對齊與讀值正確。

---

### 2.2 Drone_FC_Manual（階段 2：手動油門）

| 項目 | 內容 |
|------|------|
| **目的** | 驗證無線通訊與 4 顆馬達接線、轉向 |
| **硬體** | 階段 1 + NRF24L01+PA/LNA + ESC × 4 |
| **功能** | 接收搖桿訊號、4 顆馬達「同油門」轉動、失聯保護 |
| **不含** | PID 自穩（所以**不會飛**）|

**用途說明：**
在感測器驗證後，加入無線遙控與動力。此版本 4 顆馬達**轉速相同**，
不做姿態修正，因此無法穩定飛行，僅用於驗證：
- NRF24 通訊正常（搖桿數值有傳到）
- 4 顆 ESC 接線正確
- 馬達轉向符合 X 型配置（對角同向、相鄰反向）

⚠️ 此階段**絕對不裝螺旋槳**。

---

### 2.3 Drone_FC_PID（階段 3：PID 自穩）★ 主力版本

| 項目 | 內容 |
|------|------|
| **目的** | 飛機能自我穩定、可實際飛行 |
| **硬體** | 同階段 2 |
| **功能** | PID 自穩、X 型馬達混控、序列埠即時調參、aux2 觸發校正 |

**用途說明：**
加入飛控核心——PID 控制器。透過比較「搖桿期望姿態」與
「MPU6050 實測姿態」，計算修正量並分配給 4 顆馬達，
使飛機保持水平或依搖桿傾斜。

**核心技術：**
- **Roll / Pitch**：角度模式 PID（搖桿給期望角度）
- **Yaw**：角速度模式 PID（無磁力計，只控旋轉速度）
- **X 型混控公式**：
  ```
  M1 (前左) = 油門 + pitch + roll - yaw
  M2 (前右) = 油門 + pitch - roll + yaw
  M3 (後左) = 油門 - pitch + roll + yaw
  M4 (後右) = 油門 - pitch - roll - yaw
  ```

**特色功能：**
- 開機自動校正陀螺儀偏移 + 水平基準
- 序列埠即時調 PID 參數（kp/ki/kd/ypp/yi），不必重燒
- aux2 開關觸發重新校正
- 失聯 1 秒自動解除武裝

**已調定參數：** Kp=2.0、Ki=0.02、Kd=0.5（Roll/Pitch）、Kp_y=1.5、Ki_y=0.02（Yaw）

這是目前可實際試飛的版本。

---

### 2.4 Drone_FC_Full（階段 4：完整版）

| 項目 | 內容 |
|------|------|
| **目的** | 加入定位、定高、避障感測器，邁向自動飛行 |
| **硬體** | 階段 3 + BMP280 + GPS + VL53L0X |
| **功能** | PID 全部 + 三組感測器（旗標啟用）|

**用途說明：**
在 PID 自穩基礎上，加入三組進階感測器，用 `ENABLE_xxx` 旗標
逐個啟用（接好哪個就開哪個），避免未接的感測器卡住程式：

```cpp
#define ENABLE_BMP280   0   // 氣壓計（定高）
#define ENABLE_GPS      0   // GPS（定位、自動飛行）
#define ENABLE_VL53L0X  0   // 雷射測距（避障）
```

**三組感測器用途：**
| 感測器 | 介面 | 未來功能 |
|--------|------|---------|
| BMP280 | I²C | 自動維持高度 |
| GY-NEO6MV2 GPS | UART | 飛到指定座標、自動返航 |
| VL53L0X × 2 | I²C（透過 TCA9548A）| 前後障礙偵測、自動煞車 |

目前感測器只讀值、印序列埠，**尚未整合進飛行控制邏輯**
（定高、導航、避障為後續開發）。

## 3. 地面端程式

### Ground_TX（發射器）

| 項目 | 內容 |
|------|------|
| **硬體** | Arduino Nano + NRF24L01+PA/LNA + 雙搖桿 + 開關 |
| **功能** | 讀搖桿、打包指令、透過 NRF24 發送給飛機 |

沿用原本的 6 通道發射器電路，傳送油門、Roll、Pitch、Yaw、
兩個開關（aux1 武裝、aux2 校正）。NRF24 設定須與飛機端一致
（pipe `0xABCDABCD71`、channel 100、250kbps）。

---

## 4. 感測器獨立測試程式

開發過程中，每個感測器先用獨立程式驗證「沒壞、接線對」，
再整合進飛控。這些程式燒在不同板子上：

| 程式 | 測試對象 | 燒在哪 |
|------|---------|--------|
| **BMP280_Test** | BMP280 氣壓計 | LOLIN D32 |
| **GPS_Test** | GY-NEO6MV2 GPS（含衛星 SNR + 藍牙輸出）| LOLIN D32 |
| **ESP32CAM_Test** | ESP32-CAM 相機（AP 模式）| ESP32-CAM |
| **ESP32CAM_Flight** | ESP32-CAM 飛行版（Station 模式 + 即時調畫質）| ESP32-CAM |

## 5. 系統架構

```
┌─────────────── 無人機 ───────────────┐
│  LOLIN D32（飛控）                    │
│   ├ MPU6050（姿態）                   │
│   ├ BMP280（氣壓，預留）              │
│   ├ GPS（定位，預留）                 │
│   ├ VL53L0X × 2（避障，預留）         │
│   ├ NRF24（接收遙控）                 │
│   └ ESC × 4 → 無刷馬達 × 4            │
│                                       │
│  ESP32-CAM（影像，獨立 WiFi）         │
└───────────────────────────────────────┘
        ▲ 2.4G NRF24        ▲ WiFi
        │                   │
┌───────┴────────┐   ┌──────┴──────┐
│ Arduino Nano   │   │   電腦/手機  │
│ + NRF24 + 搖桿 │   │  看影像/AI  │
└────────────────┘   └─────────────┘
```

## 7. 後續開發方向

4. **地面顯示**：LCD 2004A 顯示遙測資料
5. **語音控制**：電腦端 Whisper 語音 → 指令
