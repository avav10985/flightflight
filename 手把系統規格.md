# 手把系統規格（ESP32 主力地面站）

> 對象：`Ground_TX_ESP32`（38-pin ESP32）。Nano 備援版維持原樣不動。
> **狀態：設計已定案，待實作。**
> 飛機端對應程式：`Drone_FC_PID`（封包與武裝邏輯要一起改）。

---

## 1. 這次改動目標

1. **油門搖桿拆彈簧** → 純油門，停在放手位置（像真遙控器，不回中）
2. **yaw 移到 L/R 肩鍵** → 原地左右旋轉（因為左搖桿拆彈簧後水平軸也不回中，yaw 不能再放那）
3. **兩個 3 段開關組合 9 模式**（00~22）
4. **底部小按鈕 + 右搖桿** → 在手把上調 PID 參數
5. **開機安全**：必須先把模式開關切到 00 才解鎖

---

## 2. 控制配置

### 飛行時
| 輸入 | 功能 |
|------|------|
| 左搖桿 上下（無彈簧） | 油門 0~100%（停在放手位置） |
| 左搖桿 左右 | 不用（yaw 已移到肩鍵） |
| 右搖桿 上下 | pitch |
| 右搖桿 左右 | roll |
| 肩鍵 L | yaw 左旋（按著 = 固定速率） |
| 肩鍵 R | yaw 右旋 |
| 3 段開關 A、B | 選模式 00~22 |

### 選單時（PID 調參）
| 輸入 | 功能 |
|------|------|
| 右搖桿 上下左右（有彈簧、會回中） | 移動游標 |
| ＋ 鈕 / − 鈕 | 調整數值 |
| 確定 鈕 | 存值 / 進下一層 |
| 返回 鈕 | 上一層 / 離開選單 |

> **進選單**：只在 mode 00（安全模式）按「確定」進入；「返回」離開。飛行模式中不能進選單。

## 3. 9 模式表

模式編號 = 開關A 檔位 × 3 + 開關B 檔位（上=0 / 中=1 / 下=2）：

| 開關 (A,B) | 模式碼 | 名稱 | 本次實作 |
|:--:|:--:|------|:--:|
| 0,0 | 0 | **校準 / 安全**（不能飛，馬達 disarm，做陀螺儀校正） | ✅ |
| 0,1 | 1 | **手動（自穩）**：角度自穩，搖桿給期望傾角、放手回平，無自動輔助 | ✅ |
| 0,2 | 2 | 〔預留〕定高（需 BMP280） | ⏳ |
| 1,0 | 3 | 〔未設定〕定點（需 GPS） | ⏳ |
| 1,1 | 4 | 〔未設定〕自動返航 RTL（需 GPS） | ⏳ |
| 1,2 | 5 | 〔未設定〕自動航線 AUTO（需 GPS） | ⏳ |
| 2,0 | 6 | 〔未設定〕自動降落 LAND | ⏳ |
| 2,1 | 7 | 〔未設定〕無頭模式 HEADLESS | ⏳ |
| 2,2 | 8 | 〔未設定〕保留 | ⏳ |

> **自穩 PID 在所有可飛模式都強制開啟**——多旋翼沒它飛不起來。
> 各「預留」模式只是在自穩之上疊加感測器輔助，感測器整合後才實作。
> **未實作模式（2~8）**：切到這些檔位 → 飛機當作未設定、拒絕武裝；地面站 LCD 顯示「模式未設定」。
> **校準**：進入 mode 00 自動執行一次陀螺儀校正（請保持飛機平放靜止）。

## 4. 開機安全流程（像真遙控器）

```
上電
 └→ 飛機進入 LOCKED（即使開關不在 00 也不解鎖、不武裝）
     └→ 把模式開關切到 00（校準/安全）一次 → 解除 LOCKED
         └→ 切到 01+ 模式、且油門在底 → 才允許 armed
```

- mode 00 → 永遠 disarmed
- 失聯 > 1 秒 → 強制 disarmed（沿用現有）
- 任何時候切回 00 → 立即 disarmed

## 5. 接腳分配（ESP32 主力 TX）

| 功能 | GPIO | 方式 | 備註 |
|------|:--:|------|------|
| 油門（左上下，無彈簧） | 34 | ADC | 不變 |
| pitch（右上下） | 32 | ADC | 不變 |
| roll（右左右） | 33 | ADC | 不變 |
| 模式開關 A（3 段） | 36 (VP) | ADC 3 階 | 間隔大、最穩 |
| 模式開關 B（3 段） | 39 (VN) | ADC 3 階 | |
| 選單鈕 ＋/−/確定/返回 | 35 | ADC 4 鈕電阻階梯 | |
| 肩鍵 L / R（yaw） | 13 / 14 | 數位 INPUT_PULLUP | yaw 要靈敏，不走階梯 |
| NRF24 | 4/5/18/23/19 | SPI | 不變 |
| LCD 2004A | 21 / 22 | I²C | 不變 |
| ~~SW_ARM 25 / SW_MENU 26~~ | — | 空出 | 武裝改由模式控制 |
| 左搖桿左右（原 J_YAW 35） | — | 空出 | yaw 移到肩鍵；35 改作選單鈕階梯 |

### 電阻階梯參考值（4 個選單鈕共用 GPIO35）
節點對 GND 接 10kΩ 下拉，各鈕由 3V3 經不同電阻接到節點：

| 鈕 | 串聯電阻 | 節點電壓 | ADC(12-bit) 約 |
|----|:--:|:--:|:--:|
| ＋ | 1kΩ | 3.0V | ~3720 |
| − | 4.7kΩ | 2.24V | ~2780 |
| 確定 | 10kΩ | 1.65V | ~2050 |
| 返回 | 22kΩ | 1.03V | ~1280 |
| 都沒按 | — | 0V | ~0 |

軟體取「最接近的已知階」判斷，數值依實測微調；同時按兩顆會落在未知區間 → 忽略。

### 3 段開關（A 接 GPIO36 為例）
common → ADC 節點，節點同時 10kΩ 接 3V3、10kΩ 接 GND（開路=中檔≈1.65V）；
上檔把 common 接 3V3（≈3.3V）、下檔接 GND（≈0V）→ 三個間隔很大的電壓階。

## 6. NRF24 封包改動

兩端共用結構要一起改（兩支 .ino 沒共用標頭，**改完必須同時重燒兩邊**，否則結構對不上會收到亂碼）。

```c
// 地面 → 飛機
struct Signal {
  byte  throttle, pitch, roll, yaw;  // yaw 由肩鍵在 TX 端算成 127±固定量
  byte  mode;        // 0~8（00=校準/安全）
  byte  flags;       // bit0=校準觸發；其餘保留
  byte  paramID;     // 0=無、1=Kp…5=Ki_y
  float paramVal;
};
```
- 移除舊的 `aux1 / aux2`（武裝改看 `mode`，校正改看 `flags` / 模式 00）
- `yaw`：TX 端 `肩鍵L→42、肩鍵R→212、都沒按→127`（對應 ±60°/s；飛機端 MAX_YAWRATE=90 讀法不變）
- `Telemetry`（飛機→地面）不變

## 7. 飛機端（Drone_FC_PID）要改的地方

| 函式 | 改動 |
|------|------|
| `struct Signal` | 同步成上面（加 `mode`/`flags`、去 `aux1/aux2`） |
| `checkArm()` | 改成：mode==0 → disarm；mode 1~8 + 油門低 + 已解除開機 LOCK → arm |
| 開機 LOCK | 新增 `safetyReleased` 旗標，開機 false，看到 mode==0 才轉 true，否則拒絕 arm |
| `checkCalibrationTrigger()` | 改成：mode==0 + `flags` 校準位邊緣 → 校正（取代 aux2） |
| `pidControl()` / `writeMotors()` | **不變**（mode 1 用現有角度自穩） |
| mode 2~8 | 視為未設定：拒絕武裝；TX LCD 顯示「模式未設定」 |

## 8. ✅ 已定案（2026-05-29）

1. **模式 01** = 角度自穩 + 無自動輔助（= 現有程式行為），不是無自穩 ACRO。
2. **校準觸發** = 進入 mode 00 自動校正一次。
3. **選單進入** = 只在 mode 00 按「確定」進入。
4. **mode 2~8 未實作時** = 視為未設定，拒絕武裝，LCD 顯示「模式未設定」。
5. **肩鍵 yaw 速率** = 固定 ±60°/s。
6. **模式編號** = 開關 A×3 + B（00=0 … 22=8），名稱照第 3 節表。
