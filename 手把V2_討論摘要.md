# 手把 V2 設計討論摘要（2026-05-29）

> 給新對話讀：這份是 2026-05-29 跟舊對話討論 V2 手把硬體規劃的結論摘要。
> 之前的 `手把系統規格.md` 與 `手把接線總表.md` 是 V1（30-pin WROOM + LCD 2004A）的版本，
> 使用者現在決定**跳過 V1、直接做 V2**，本檔記錄 V2 規劃的最新決定。

---

## 1. 使用者已備齊的硬體

| 元件 | 規格 | 備註 |
|------|------|------|
| **ESP32-S3 開發板** | ESP32-S3 N16R8（44 隻腳）| **16 MB Flash + 8 MB OCTAL PSRAM**；雙 USB-C（一個 UART bridge、一個原生 USB）；IPEX 外接天線 |
| **TFT 螢幕** | **MAR2406**（MCUFRIEND 系列）| 2.4"、320×240、**ILI9341 並列 8-bit**、含觸控（XPT2046）、板上有 SD 卡座 |
| **SD 卡模組** | 另外買的獨立模組 | 也可改用 TFT 板上的 SD 卡座，效果一樣 |
| **I²S 語音** | INMP441 麥克風 + MAX98357A 放大 + 小喇叭 | 已買 |
| 既有沿用品 | 搖桿×2、NRF24+PA/LNA、100µF 電容、MTS-103 ×2、輕觸鈕、電阻組、buck、2S 電池 | 同 V1 |

使用者習慣**把主控與螢幕做成插拔座**（方便程式燒錄時取下）。

---

## 2. N16R8 的關鍵限制

**OCTAL PSRAM 吃掉 GPIO 33~37（永久佔用）**，加上其他內部佔用：

| GPIO | 狀態 |
|------|------|
| 0 | 可用（小心：BOOT strapping，開機要 HIGH）|
| 1~18, 21 | ✅ 可用 |
| 19, 20 | ⚠️ 原生 USB（如改用 UART USB-C debug 就釋出）|
| 26~37 | ❌ 內部 Flash + OCTAL PSRAM 佔用 |
| 38~42 | ✅ 可用 |
| 43, 44 | ⚠️ UART0（如改用原生 USB debug 就釋出）|
| 45, 46 | ❌ strapping，避開 |
| 47, 48 | ✅ 可用 |

**實際可用 28~30 隻**（取決於用哪條 debug 路線）。

---

## 3. 已定案的設計決定

1. **手把不做電池監測**（省 1 隻腳；之後 LCD 變暗自己會發現要充電）
2. **LCD_RST 接 3V3 + 10k 上拉**（不接 GPIO，省 1 隻腳）
3. **SD 卡與 NRF24 共用 SPI 匯流排**（只多 1 隻 CS）
4. **採分階段：V2-A 先做、V2-B 之後加 I²S 語音**

理由:分階段先讓 V2-A 跑通,V2-B 加語音時把 I²S 4 隻腳指派到「MSP2806 SPI 換來的釋出腳 11~14」,
**兩條 debug 序列埠都保留**(19/20 native USB + 43/44 UART0)。
原本「擇一犧牲」的計畫是 MAR2406 並列年代訂的,SPI 換掉後就不必再犧牲了。

---

## 4. V2-A 完整腳位分配（**現在要做的**）

27 隻腳：

| GPIO | 用途 | 備註 |
|:--:|------|------|
| 1 | 油門（左搖桿上下，無彈簧）| ADC1 |
| 2 | **左搖桿 X(可選 yaw 來源)**| ADC1_CH1;預設不啟用,選單切換 |
| 4 | roll（右搖桿左右）| ADC1 |
| 5 | SW_A 模式開關 A（3 段）| ADC1 3 階 |
| 6 | SW_B 模式開關 B（3 段）| ADC1 3 階 |
| 7 | MENU_BTN 選單鈕電阻階梯（+、−、OK、Back）| ADC1 4 階 |
| 8 | SHOULDER_L 肩鍵左（預設 yaw 左）| 數位 INPUT_PULLUP |
| 9 | SHOULDER_R 肩鍵右（預設 yaw 右）| 數位 INPUT_PULLUP |
| **10** | **右搖桿 pitch(Y 軸)** | **ADC1_CH9** |
| 11~18 | TFT 並列資料 D0~D7 | 8 隻一組 |
| 21 | TFT RS (DC) | |
| 47 | TFT WR | |
| 48 | TFT CS | |
| LCD_RST | 接 3V3 + 10k 上拉 | 不佔 GPIO |
| LCD_RD | 接 3V3 | 不讀，省事 |
| 38 | NRF24 SCK + SD SCK 共用 | SPI2 |
| 39 | NRF24 MOSI + SD MOSI | SPI2 |
| 40 | NRF24 MISO + SD MISO | SPI2 |
| 41 | NRF24 CE | |
| 42 | NRF24 CSN | |
| 0 | SD_CS（10k 上拉）| BOOT 腳，開機 HIGH 安全 |

**V2-B I²S 4 隻指派**:**GPIO 11/12/13/14**(BCLK/WS/DOUT/DIN),用 MSP2806 SPI 換來的釋出腳。
**Debug 序列埠都保留**:GPIO 19/20(native USB CDC)+ 43/44(UART0),兩條都活,V2-B 加完還能燒 + 看 Serial。
**真正釋出可用**:15/16/17/18/47(共 5 隻給 V2-C/D)。
**未用**：GPIO 3（避開）、45、46（strapping）、26~37（內部）。

---

## 5. V2-A 功能範圍

V2-A 已可做：
- TFT 彩色顯示（飛行畫面、選單）
- 雙向 NRF24 遙測（飛機端已有 ACK Payload 支援）
- 9 模式（兩個 3 段開關組合）
- 4 鈕電阻階梯選單操作
- 油門無彈簧、yaw 肩鍵
- SD 卡儲存設定 / log

V2-A **暫不做**：
- I²S 語音（V2-B 再加）
- 觸控螢幕（如果想加，要再挪 5 隻腳；暫不接）

---

## 6. V2-B(之後)的 I²S 4 隻腳

加 I²S 麥克風 + 喇叭時:

1. **I²S 4 隻腳改用釋出腳**(2026-06-03 修正,不再犧牲 debug):
   - **GPIO 11 = BCLK**(共用 INMP441 SCK + MAX98357A BCLK)
   - **GPIO 12 = WS/LRCLK**(共用 INMP441 WS + MAX98357A LRC)
   - **GPIO 13 = DOUT**(到 MAX98357A DIN)
   - **GPIO 14 = DIN**(從 INMP441 SD)
2. **兩條 debug 都保留**:19/20 native USB CDC + 43/44 UART0
3. INMP441 跟 MAX98357A 模組板放 A 板;喇叭(JINHUA 28mm 8Ω/2W 內磁)鎖在外殼出聲位置
4. INMP441 L/R 腳接 GND(=左聲道,單聲道用)
5. MAX98357A GAIN/SD 都浮空(=預設 9dB、mono L+R 混音)
6. MAX98357A VIN 吃 5V(從 buck OUT 拉),推 8Ω 喇叭有 ~1.8W

## 7. 飛機端封包協定（已定案）

與 V2 手把相容。飛機端不需要再改。

**接著要做的**：
1. 把 `Ground_TX_ESP32.ino` 從 V1（30-pin WROOM + LCD 2004A）改成 V2-A（S3 + MAR2406 並列 TFT）
2. 新寫 `手把接線總表_V2.md` 取代 V1 接線表
3. TFT_eSPI 函式庫設定（並列 8-bit + ILI9341）
4. 寫 V2-A UI 框架（飛行畫面、選單）
